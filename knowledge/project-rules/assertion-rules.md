# 断言规则

- API 断言至少覆盖状态码、业务码、关键字段和错误信息。
- UI 断言至少覆盖页面状态、关键文案和可操作结果。
- 断言应与需求预期绑定，不只判断请求成功或元素存在。
- 对时间、随机值、异步结果应使用稳定等待和容忍策略。

## API 断言分层

- 协议层：HTTP 状态码、响应类型。
- 业务层：业务码、错误码、错误消息。
- 数据层：关键字段、类型、取值范围。
- 状态层：错误次数、锁定状态、token 过期。

## UI 断言分层

- 可见状态：关键文案、按钮状态、跳转结果。
- 业务状态：登录态、错误提示、锁定提示。
- 稳定性：等待条件和失败截图策略。
