# 基于风险的测试

基于风险的测试用于在时间有限时优先覆盖高影响、高概率、高不可逆损失的场景。

## 风险维度

- 用户影响。
- 资金或数据影响。
- 安全与合规影响。
- 历史缺陷密度。
- 技术复杂度和依赖不稳定性。

## 输出建议

每个风险项应给出风险等级、覆盖策略、必要用例和是否需要人工确认。

## 示例模板

| 风险 | 等级 | 原因 | 覆盖策略 | 人工确认 |
|---|---|---|---|---|
| 锁定规则失效 | 高 | 影响账号安全 | P0 用例 + API 自动化草稿 | 是 |
| token 过期未拦截 | 高 | 影响会话安全 | P0 用例 + 受保护接口验证 | 是 |

## 常见误区

- 平均分配测试精力。
- 忽略安全、权限和数据一致性风险。
