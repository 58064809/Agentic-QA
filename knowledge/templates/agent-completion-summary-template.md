# Agent 标准完成回执模板

## 使用说明

完成仓库任务后使用本模板回复。只写摘要、路径、验收状态和待确认项，不粘贴完整文件或完整 diff。

## 变更摘要

- 简要说明本次完成了什么，不超过 3~5 条。

## 修改文件

| 文件 | 说明 |
|---|---|
| `path/to/file.md` | 说明该文件变更点 |

## 验收结果

| 命令 | 结果 | 说明 |
|---|---|---|
| `command` | 通过/失败/未执行 | 未执行命令必须说明原因；失败命令必须说明失败摘要和处理方向 |
| GitHub Actions `CI` | 通过/失败/未触发/未看到结果 | 说明触发和结果状态；未看到结果必须说明原因 |

## 待人工确认

- [ ] 需要人工确认的业务判断、审核门、环境授权或风险接受项。
- 如果没有待确认项，写“暂无新的待人工确认项”。

## 下一步建议

- 给出 1~3 条克制的下一步建议。
