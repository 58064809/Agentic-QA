# 接口自动化断言规则

## 断言分层

| 层级 | 断言对象 | 示例 |
|---|---|---|
| 协议层 | HTTP 状态码、响应类型、响应耗时 | `status_code == 200` |
| 业务层 | 业务 code、错误码、错误消息 | `code == SUCCESS` |
| 数据层 | 字段存在、类型、枚举、范围 | `$.data.id` 存在且为 string |
| 状态层 | 账号状态、订单状态、库存、次数 | 错误次数递增、订单状态变更 |
| 安全层 | 鉴权、权限、敏感字段 | 未登录返回 401，不返回密码 |

## 必须覆盖

- 每条用例至少包含协议层断言。
- 有业务响应体时必须包含业务层断言。
- 响应包含关键数据时必须包含数据层断言。
- 涉及状态流转、金额、库存、权限、幂等或风控时，应补充状态层断言或人工确认项。

## 不稳定字段

以下字段不建议断言固定值：

- 时间戳。
- 随机 ID。
- trace_id。
- 动态 token。
- 异步任务状态。

可改为断言存在性、格式、类型或范围。

## 禁止事项

- 不只断言 `status_code == 200`。
- 不把错误提示文案作为唯一断言。
- 不在断言中写真实敏感数据。
- 不依赖生产环境数据。
