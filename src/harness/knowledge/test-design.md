# 测试设计方法

组合使用以下方法，并把每个测试点映射到需求、规则或风险：

- 等价类：同时覆盖有效、无效、缺失、空白、格式错误和权限不足的代表值。
- 边界值：覆盖 `N-1`、`N`、`N+1`，以及最小/最大、空值、超长、时间窗口到期和恢复。
- 场景法：覆盖主流程、替代流程、中断、重试、取消、超时和跨角色流程。
- 状态迁移：明确初态、事件、目标状态、禁止迁移、重复事件和恢复事件。
- 组合与回归：对高风险参数做成对或关键组合，关联历史故障和受影响依赖。

用例步骤必须可执行，预期必须可观察或断言。每条用例聚焦一个核心目标；不明确的数据、
账号、环境和规则写入“待确认项”。固定输出 11 列，并在覆盖矩阵中提供至少一条有效映射。
