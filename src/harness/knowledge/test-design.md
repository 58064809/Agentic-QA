# 测试设计方法

## 方法矩阵

| 方法 | 覆盖 |
|---|---|
| 等价类 | 有效、无效、缺失、空白、格式错误、权限不足 |
| 边界值 | `N-1/N/N+1`、最小/最大、空值、超长、时间到期与恢复 |
| 场景法 | 主/替代流程、中断、重试、取消、超时、跨角色 |
| 状态迁移 | 初态、事件、目标态、禁止/重复/恢复事件 |
| 组合与回归 | 高风险参数组合、历史故障和受影响依赖 |

## 输出契约

- 每个测试点映射需求、规则或风险；用例聚焦一个核心目标。
- 步骤可执行，预期可观察或断言；不明确内容进入“待确认项”。
- 固定 11 列，每条 Markdown 记录一个物理行，多步骤使用 `<br>`。
- 覆盖矩阵至少包含一条有效映射。

没有 OpenAPI、数据字典或系统设计证据时，不得编造接口、数据库表或字段。
