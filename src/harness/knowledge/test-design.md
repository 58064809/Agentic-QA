# 测试设计方法

## 方法矩阵

| 方法 | 覆盖 |
|---|---|
| 等价类 | 有效、无效、缺失、空白、格式错误、权限不足 |
| 边界值 | `N-1/N/N+1`、最小/最大、空值、超长、时间到期与恢复 |
| 场景法 | 主/替代流程、中断、重试、取消、超时、跨角色 |
| 状态迁移 | 初态、事件、目标态、禁止/重复/恢复事件 |
| 组合与回归 | 高风险参数组合、历史故障和受影响依赖 |

## 输出契约

- 先以需求分析和冻结来源建立原子覆盖清单：每条 confirmed 规则、配置行、枚举值、边界、
  状态迁移和高风险推断都必须有明确处置。
- 每个测试点映射需求、规则或风险；用例聚焦一个核心目标，失败时能定位到具体规则。
- 步骤可执行，预期可观察或断言；不明确内容进入“待确认项”。
- 固定 11 列，每条 Markdown 记录一个物理行，多步骤使用 `<br>`。
- 配置表、档位、枚举和对应关系逐项覆盖；数值阈值至少考虑阈值前、阈值和阈值后。
- 覆盖矩阵列出所有原子规则/风险及实际用例 ID，不得用范围缩写或“待补充”占位。
- 用例数量由规则和风险决定，不以任意条数截断；相同前置和动作可采用表驱动，但每个配置值
  仍须在测试数据和覆盖矩阵中可见。

没有 OpenAPI、数据字典或系统设计证据时，不得编造接口、数据库表或字段。
