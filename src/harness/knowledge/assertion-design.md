# 断言与证据设计

## 断言矩阵

| 类型 | 必须验证 |
|---|---|
| API | 协议状态、业务码、关键字段类型/值域、错误语义、持久化状态与副作用 |
| UI | 关键文案、控件状态、导航结果、业务状态、稳定等待与失败截图 |
| 异步 | 可观测完成条件、合理超时与轮询边界，不使用固定睡眠 |
| 动态数据 | 从响应或 fixture 提取 Token、业务 ID；真实凭证只读环境变量 |

## 证据要求

- 记录来源、观察时间、执行环境和关联步骤。
- 断言绑定业务预期，不只写“请求成功”或“元素存在”。
- 无法取得证据时标记未执行或待确认，不把预期写成实际结果。
