# 需求分析草稿：5月活动玩法

## 1. 需求背景与目标

- **背景**：PRD 提出“脆皮打工人养护中心”H5活动，旨在通过趣味测试和互动玩法，覆盖当前职场人群（“脆皮打工人”）的痛点，降低参与门槛，引导用户参与并分享。
- **业务目标**：
  - 活动曝光与用户参与：通过“脆皮打工人”测试吸引用户参与。
  - 社交裂变：用户生成诊断书后可保存/分享，并可通过“邀请好友”机制带动新用户。
  - 平台导流与留存：将活动流量引导至平台职场圈、治愈圈、专业咨询等核心功能板块，实现长期留存。
  - 提升互动：通过话题区“会诊”评论互动和运营号带动，提升用户活跃度。

## 2. 业务范围

### 范围内能力
- **测试功能**：用户完成“脆皮打工人”10道选择题测试（每题4个选项），系统自动根据评分规则（选项分值A=4,B=3,C=2,D=1）计算总分（10-40分），并输出对应等级（轻度/中度/重度脆皮）的诊断结果。
- **诊断书生成与分享**：用户完成测试后，生成专属电子诊断书（含等级、结果、症状、医嘱、平台logo），支持用户保存和分享（预期为图片形式）。
- **邀请好友（裂变）**：老用户生成专属邀请链接/海报，新用户通过链接/海报注册并参与活动（完成测试+发布1条互动内容），老用户获得奖励。
- **活动话题区互动**：用户可将诊断书发布到活动话题区，其他用户可进行评论（“会诊”），提供评论互动功能。
- **专业导流**：活动页设置“专业养护通道”入口，引导用户进入专业圈、治愈圈等；话题区置顶相关圈子入口。
- **奖励体系**：
  - 参与奖：完成测试即可获得“脆皮打工人”专属头像框（有效期15天）。
  - 互动奖：评论区随机抽取50名用户获得终极养护礼包。
  （来源：PRD 奖励体系部分）

### 范围外能力
- 不包含其他独立活动（如“给室友开个方子”、“定时红包活动”），这些活动在PRD中仅有标题但无详细内容，不计入本分析范围。
- 不包含后台管理功能（如头像框发放、礼包发放、数据统计等系统功能），PRD未提及后台实现细节。
- 不包含平台其他非活动相关页面（如个人主页、设置等）。
- 不包含支付、订单等交易模块。

## 4. 主流程拆解

### 主流程1：用户参与脆皮测试
1. 用户进入H5活动页面（未登录 -> 提示登录/自动跳转登录）。
2. 用户查看活动介绍、宣传图、Slogan（“拒绝内卷，在线会诊”）。
3. 用户点击“开始测试”（触发测试流程）。
4. 按顺序展示10道题目（每题4个选项，单题单选）。
5. 用户选择每题答案后，点击“下一题”或自动跳转下一题（交互方式需确认）。
6. 完成10题后，系统根据评分规则计算总分，匹配等级（轻度/中度/重度脆皮）。
7. 展示诊断结果（等级、诊断书、症状、医嘱、平台logo等）。
8. 用户可执行操作：保存诊断书图片、分享、发布到话题区。

### 主流程2：用户生成诊断书并分享
1. 用户完成测试，获得诊断结果页面。
2. 页面提供“保存”按钮（保存为图片）和“分享”功能（原生分享/海报分享）。
3. 用户点击分享，可将诊断书/活动链接分享至微信、朋友圈等平台。

### 主流程3：用户参与互动（话题区）
1. 用户点击“发布”按钮（可能在诊断结果页/话题区），输入配文（如“求会诊”“求药方”），发布互动内容。
2. 发布后，其他用户（普通用户、运营号）可以在该内容下评论（“会诊”）。
3. 运营号带头进行玩梗评论，带动用户互动。

### 主流程4：用户邀请好友（裂变）
1. 老用户进入活动页面的裂变入口（如“邀请好友”）。
2. 系统生成专属邀请链接/海报（含二维码）。
3. 老用户分享链接/海报给未注册好友。
4. 新用户通过链接/海报注册APP（进入注册流程，PRD未说明，假设与常规注册一致）。
5. 新用户完成测试并发布1条互动内容后，系统判定邀请有效。
6. 老用户获得奖励（“脆皮养护礼包”或“一次免费专业养护通道活动”）。
7. （对应的新用户可能也获得奖励？PRD未明确说明其奖励）

## 5. 分支流程与异常流程

### 分支流程
- **用户测试中途退出**：用户完成部分题目后刷新/离开/退出，再次进入活动时，应询问是否继续上次未完成测试，或从头开始？需确认。
- **用户修改答案**：提交前是否允许返回修改答案？需确认。
- **用户未发布互动内容**：仅完成测试但未发布内容，视为不满足“裂变邀请”中新用户的条件。 （来源：PRD“新用户完成脆皮测试+发布1条互动内容”）
- **用户选择“非脆皮”选项**：例如第3题第4项“毫无感觉（非脆皮）”，第8题第4项“从容解决（非脆皮）”，符合计分规则（D=1分），但诊断书内容未覆盖此类“非脆皮”人群，需确认对应结果。 （假设：系统仍按计分规则判断等级，但等级文本可能缺少对非脆皮用户的特殊处理，需确认）

### 异常流程
- **用户不满足年龄/地域限制**：PRD未说明是否有年龄或地域限制（如未成年人无法参与）。 假设无限制。
- **用户被邀请后未注册**：新用户点击邀请链接，但未完成注册流程。 系统应能追踪到该入口？ 需确认。
- **用户重复注册同一手机号**：假设通过手机号+验证码注册，应做去重校验。
- **奖励发放失败**：用户满足条件后，头像框/礼包发放失败。 需要有重发/补偿机制（接口幂等）。
- **评论内容包含敏感词**：用户发布互动/评论时，需进行敏感词过滤，避免违规内容。
- **用户举报内容**：互动区可能产生不当评论，需支持举报功能。

## 7. 数据字段与状态流转

### 核心对象：诊断书
- 包含字段：用户Nickname、用户头像、等级（轻度/中度/重度)、症状文本、结果文本、医嘱文本、对应等级的症状图片、平台logo。
- 状态：生成（用户完成测试后立即生成）。

### 核心对象：邀请关系
- 状态：待激活（新用户点击链接但未注册）、已激活（新用户完成测试+互动）、已奖励（老用户已领取奖励）。
- 字段：邀请人ID、被邀请人ID、邀请时间、激活时间、奖励发放状态。

### 状态流转（假设）
- 邀请关系：待激活 -> 已激活 -> 已奖励 （不可逆向）。
- 奖励领取：未领取 -> 已领取。 （可能有时效，需确认）

## 9. 测试范围建议

| 测试模块 | 优先级 | 测试方向 |
| :--- | :--- | :--- |
| **脆皮测试功能** | P0 | 正向：所有题目组合、计分规则验证、等级边界值测试（10-15/16-25/26-40）、异常值输入、超时未提交、中途退出、修改答案逻辑 |
| **诊断书生成** | P0 | 正确性：等级显示文本、症状、医嘱、logo。边界：满分/最低分。异常：测试结果生成失败处理 |
| **裂变邀请** | P0 | 正向：正常邀请流程、解锁邀请按钮。逆向：未登录用户无法邀请、邀请链接失效。并发：多人同时邀请、同一新用户被多次邀请 |
| **奖励发放** | P0 | 正向：满足条件后头像框/礼包到账。异常：发放失败、重复领取、有效期校验、15天后头像框消失 |
| **话题区互动** | P1 | 正常发布/评论、异常评论（敏感词、长文本、特殊字符）、运营号特权、搜索/排序功能 |
| **导流跳转** | P1 | 点击“专业养护通道”进入对应圈子、点击圈子入口正常跳转、跳转后返回到活动页 |
| **兼容性** | P2 | H5页面在不同机型/浏览器/微信内打开正常、图片分辨率适配 |
| **安全/合规** | P1 | 敏感词过滤、用户信息保护（隐私政策、授权）、邀请链接防刷 |
