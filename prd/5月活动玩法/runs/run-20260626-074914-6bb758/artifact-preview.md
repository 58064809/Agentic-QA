# 需求分析草稿

## 1. 需求背景与目标

- **需求背景**：基于 PRD “5月活动玩法” 中的“脆皮打工人养护中心”活动。主题为“拒绝内卷，在线会诊”，定位是面向打工人的趣味互动测试。
- **业务目标**：
    - **核心目标**：通过趣味测试、诊断书生成与分享、互动会诊，吸引用户参与，实现活动页面的传播与拉新。
    - **裂变目标**：通过“邀请好友，双向有礼”机制，带动新用户注册并参与活动。
    - **导流目标**：将活动流量导流至平台的“职场圈”、“治愈圈”、“专业咨询”等核心板块，实现用户留存。
    - **互动目标**：通过官方运营号带头评论，营造“会诊”氛围，提升用户评论活跃度。
- **风险假设**：假设用户对此类玩梗文案的接受度较高，且愿意在社交平台分享。

## 2. 业务范围

- **范围内能力**：
    - 用户参与 H5 互动测试，完成10道趣味选择题。
    - 根据 AI 算法自动计算总分，并生成对应的“专属电子诊断书”。
    - 诊断书包含玩梗文案，并可生成带平台 logo 的图片，支持保存与社交分享。
    - 用户在活动话题区发布内容，其他用户可进行评论“会诊”。
    - 用户通过邀请链接/海报邀请新用户，并获取相应奖励。
    - 活动页面提供“专业养护通道”和“圈子”入口，实现导流。
    - 用户完成测试后获取“脆皮打工人”专属头像框（有效期15天）。
    - 平台在评论区随机抽取用户发放奖励。
- **范围外能力**（基于现有文本，非确认）：
    - 测试题目、答案选项、评分规则的动态管理（假设为固定配置）。
    - 头像框的具体使用场景（仅限 APP 内使用，数量、样式等未明确）。
    - “专业养护通道”、“圈子”、“主理人联动”的详细交互流程。
    - “邀请好友”功能的后台数据统计及奖励发放的具体规则。

## 3. 角色与权限

- **普通用户**：
    - **活动参与**：完成测试、查看诊断书、发布活动内容、评论他人内容、点赞。
    - **裂变**：生成专属邀请链接/海报，邀请新用户。
    - **导流**：点击“专业养护通道”及“圈子”入口。
    - **领取奖励**：获取头像框、参与评论抽奖、获取邀请奖励。
- **新用户**：通过邀请链接或海报注册并完成活动。
- **活动管理员/运营**：
    - **内容管理**：发布和置顶活动话题，带头评论互动。
    - **用户管理**：抽取评论获奖用户。
    - **活动配置**：配置测试题目、规则、奖励等（需确认后台有无管理界面）。
- **平台主理人**：发布专业建议，发起线上茶话会。
- **系统/服务**：
    - 自动计算测试分数并生成诊断结果。
    - 生成邀请链接/海报。
    - 记录用户行为（发布、评论、邀请）并发放奖励。

## 4. 主流程拆解

1.  **用户进入H5活动页** → 查看活动介绍及测试题目。
2.  **用户答题**：依次完成10道选择题（每题默认或多个选项）。
3.  **系统自动评分**：根据PRD规则（A=4, B=3, C=2, D=1），计算总分。
4.  **系统分级**：根据总分（10-15:轻度，16-25:中度，26-40:重度）确定脆皮等级。
5.  **生成诊断书**：展示与等级对应的诊断结果、症状和医嘱文案，并生成带平台logo的图片。
6.  **用户操作**：
    - **分享**：将诊断书图片分享至社交平台。
    - **保存**：将诊断书图片保存至本地。
    - **发布**：在活动话题区发布配文内容。
    - **互动**：查看他人发布的内容，并进行评论“会诊”。
7.  **用户引导**：
    - 活动页提供“专业养护通道”、“圈子”等导流入口。
    - 用户可点击进入相关圈子或咨询主理人。
8.  **裂变邀请**（可选路径）：
    - 老用户生成邀请链接/海报。
    - 新用户通过链接/海报注册APP并参与活动。
    - 系统判定有效邀请，向老用户发放奖励。

## 5. 分支流程与异常流程

- **分支流程A：用户拒绝授权（如相册、地理位置或社交分享权限）**：
    - 保存诊断书/生成邀请海报时，系统需处理拒绝授权的情况，引导用户手动保存或提示授权。
- **分支流程B：用户未登录**：
    - 用户若未登录，点击“发布”、“互动”、“邀请好友”等功能时，应引导至登录/注册页面。完成测试本身可能不要求登录。
- **分支流程C：邀请无效**：
    - 新用户注册后未完成测试或未发布互动内容，则不视为有效邀请。
- **异常流程A：网络异常**：
    - 答题过程中断网。
    - 生成诊断书图片失败（图片需依赖服务器资源）。
    - 发布评论失败，需有重试机制或离线提示。
- **异常流程B：用户重复提交或并发邀请**：
    - 需确认邀请机制是否支持幂等，防止一个用户被多次计为被邀请人。
- **异常流程C：奖励发放失败**：
    - 头像框、奖励礼包发放失败的处理逻辑。系统需记录失败日志并支持人工补发。
- **异常流程D：分数计算异常**：
    - 用户未完整答题或选择无效值时，系统应如何处理？假设用户必须完成所有题目才能得到结果。
