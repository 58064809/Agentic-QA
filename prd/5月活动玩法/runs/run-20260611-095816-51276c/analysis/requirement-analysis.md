# 5月活动玩法 - 脆皮打工人养护中心 - 需求分析草稿

## 1. 需求背景与目标
- **背景**：基于职场打工人“脆皮”健康痛点（如久坐腰酸、精神内耗、加班疲惫），推出“脆皮打工人养护中心”H5活动，结合玩梗测试、诊断书生成、社区互动和裂变机制，提升用户活跃度和平台留存。
- **业务目标**：
  1. 用户参与并完成脆皮测试，生成专属电子诊断书（可保存、可分享）。
  2. 通过用户发布诊断书及互动评论，激活活动话题区，强化社交传播。
  3. 老用户邀请新用户完成测试+互动，实现裂变拉新。
  4. 将活动流量导流至平台职场圈、治愈圈、专业咨询板块，提升长期留存。
- **上线时间**：5月活动（具体日期待确认）。
- **Slogan**：拒绝内卷，在线会诊。

## 2. 业务范围
- **范围内**：
  - H5活动宣传页展示（含测试题目、选项、操作入口）。
  - 10道脆皮测试题展示与答题交互。
  - 选项分值设置（A/B/C/D，分别对应4/3/2/1分）。
  - AI自动计算总分并生成脆皮等级（轻度/中度/重度）。
  - 每类等级对应的专属诊断结果文案及图片（含平台logo）生成与展示。
  - 诊断书保存（截图或生成卡片）、分享（朋友圈/社群）功能。
  - 用户将诊断书发布到活动话题区（标题为“求会诊”“求药方”，配文感染职日常）。
  - 话题区用户评论互动（“会诊”“开方”等玩梗评论）。
  - 老用户生成专属邀请链接/海报，新用户注册并完成测试+发布1条互动内容视为有效邀请。
  - 老用户获“脆皮养护礼包”或“一次免费专业养护通道活动”奖励。
  - 活动页设置“专业养护通道”入口，跳转到专业圈（心理、养生、职场）。
  - 活动话题区置顶“职场摸鱼圈”“治愈放松圈”“脆皮养护交流圈”入口。
  - 主理人发起“脆皮打工人线上茶话会”引流。
  - 参与奖：完成测试得“脆皮打工人”专属头像框（有效期15天）。
  - 互动奖：评论区随机抽取50名获“终极养护礼包”。
- **范围外**（假设，需确认）：
  - 给室友开个方子 H5活动（仅提及，但未提供具体需求）。
  - 定时红包活动（仅提及，但未提供具体需求）。
  - 所有活动的后台管理功能（如用户管理、奖励配置、数据统计）。
  - 专业养护通道的第三方咨询/主理人服务对接。
  - 邀请链接/海报的生成逻辑与防刷机制。
  - 头像框的使用规则（如在哪些页面展示、是否可叠加）。
  - 终极养护礼包的具体内容与实际发放。
- **图片内容**：输入材料中包含大量图片引用（如选项图片、诊断书示例图、症状对应图片等），依据系统指令，当前Runtime不分析图片内容，所有图片相关的展示逻辑、样式、布局均未分析，需人工确认。

## 3. 角色与权限
| 角色 | 描述 | 权限范围 |
|---|---|---|
| **普通用户** | 活动目标用户，即职场打工人 | 浏览H5活动页、参与测试、查看/保存/分享诊断书、发布诊断书到话题区、评论互动（含被邀请新用户） |
| **被邀请新用户** | 通过老用户邀请链接/海报注册APP的用户 | 同普通用户，且完成测试+发布1条互动内容后可算作有效邀请 |
| **其他用户（互动者）** | 参与话题区评论的用户 | 评论、点赞（假设）、分享诊断书 |
| **运营人员（官方）** | 官方运营人员 | 带头发布评论（玩梗），管理话题区（猜测，需确认），配置奖品、审核用户发布内容（假设） |
| **平台主理人** | 心理、养生、职场圈主理人 | 在专业通道发布专业建议、发起线上茶话会 |
| **后台系统** | 自动管理测试评分、奖励发放、邀请计数 | 无直接用户交互，作为系统服务 |

- **权限差异**：普通用户与互动用户权限基本一致，新用户需完成指定行为才能触发邀请奖励。运营人员与主理人权限通常依赖后台角色配置（当前需求未明确定义）。

## 4. 主流程拆解
### 4.1 主流程一：用户参与脆皮测试
1. 用户进入H5活动宣传页。
2. 用户阅读并参与10道测试题（每题从A/B/C/D四个选项中选一个）。
3. 用户完成所有题目，系统根据A=4分，B=3分，C=2分，D=1分，AI自动计算总分。
4. 系统根据总分区间（10-15分=轻度，16-25分=中度，26-40分=重度）生成对应等级的诊断结果（含文案和症状列表）。
5. 系统展示电子诊断书（含用户等级、文案、症状，预设带平台logo的图片），并提供保存和分享入口。
6. 用户可保存诊断书图片至本地，或通过微信等社交渠道分享。
### 4.2 主流程二：用户互动与社区参与
1. 用户在活动话题区（或分享页面跳转）发布自己的诊断书，配文“求会诊”“求药方”或自己的职场脆皮日常。
2. 其他用户浏览该动态，在评论区进行“会诊”（如“确诊重度脆皮，建议立刻请假躺平”）。
3. 运营人员带头评论（玩梗语气），提升话题热度。
### 4.3 主流程三：老用户邀请新用户（裂变）
1. 老用户（已完成测试）在活动页面获取专属邀请链接/海报。
2. 新用户通过微信或H5分享，注册APP并参与活动（完成测试+发布1条互动内容）。
3. 系统记录有效邀请，自动发放老用户奖励（脆皮养护礼包或一次免费专业养护通道活动）。
### 4.4 主流程四：导流至平台核心功能
1. 活动页设置“专业养护通道”入口，用户点击后跳转至专业圈（心理、养生、职场主理人建议）。
2. 活动话题区置顶三个圈子入口（职场摸鱼圈、治愈放松圈、脆皮养护交流圈），用户可点击进入，参与后续社交。
3. 主理人发起线上茶话会，用户可通过活动页广告（假设）报名或参与。

## 5. 分支流程与异常流程
- **分支流程**：
  - **用户选择“活力满满（异类）”**：得分较低，属于轻度脆皮的其他选项，正常评分。
  - **用户分享诊断书到朋友圈/群**：分享成功后，用户可返回H5或自动打开APP（假设）。
  - **邀请海报/链接生成失败**：用户刷新或通过客服重新获取（假设）。
  - **新用户注册流程中断（如填写信息未完）**：邀请不计数，用户需重新完成注册+测试+互动。
  - **话题区评论频繁/内容违规**：是否需要业务规则限制或内容审核（待确认）。
  - **主理人茶话会人数限制**：超过限制时，用户无法加入（假设）。
- **异常流程**：
  - **题目未全部作答便提交**：系统提示“请完成所有题目”，不得计算分数。
  - **同一用户多次参与测试**：
    - *假设（待确认）*：同一用户是否允许重复答题？若允许，诊断书是否更新？是否影响互动（第一条动态覆盖？）
  - **邀请链接失效/过期**：新用户打开链接提示“活动已结束”或“链接无效”。
  - **高分或低分异常边界值（总分<10或>40）**：系统应能处理，但规则未定义对应等级。假设取整/错误隐藏。
  - **网络异常/页面加载失败**：用户重新进入H5，数据需保留（状态保存或重新开始，待确认）。
  - **并发用户大量访问**：活动页响应缓慢，需确认后端负载能力。
  - **分享图片生成失败**：提示用户“保存失败，请重试”或提供文字保存方案。
  - **用户拒绝保存/分享**：无影响，正常返回活动主页。
  - **后台奖励发放失败**：需降级处理，如失败记录日志，重试或人工干预。

## 6. 业务规则清单
1. **分值规则**：A=4分，B=3分，C=2分，D=1分。
2. **等级规则**：总分10-15分=轻度脆皮，16-25分=中度脆皮，26-40分=重度脆皮。总分超出此区间（<10或>40）无定义。
3. **诊断书内容规则**：
   - 轻度脆皮：诊断结果=“职场边缘脆皮，偶尔摆烂，偶尔努力”；症状=“久坐腰酸、偶尔emo、周末必补觉”；医嘱=“少加班，多摸鱼，咖啡少喝，保温杯常带”。
   - 中度脆皮：诊断结果=“标准脆皮打工人，主打一个‘表面坚强，内里崩塌’”；症状=“熬夜成瘾、浑身酸痛、精神内耗、不想上班”；医嘱=“强制躺平，拒绝内卷，每天睡够8小时，少看工作消息”。
   - 重度脆皮：诊断结果=“重症脆皮患者，急需养护，建议原地退休”；症状=“一碰工作就累、焦虑缠身、腰酸背痛、emo常态化”；医嘱=“全程摆烂，不看工作群，不接工作电话，优先养护自己，必要时找平台专业心理/养生博主咨询”。
4. **诊断书保存/分享规则**：诊断书可直接生成图片（带平台logo），用户可保存或分享。
5. **发布动态规则**：用户发布诊断书时，可自行配文（“求会诊/求药方”或随意感染职日常）。发布时间无限制。
6. **裂变邀请规则**：
   - 老用户通过专属链接/海报邀请新用户，新用户注册APP并完成测试+发布1条互动内容，即视为有效邀请。
   - 每个老用户有效邀请次数、奖励上限（待确认）。
   - 新用户同1人通过多个链接注册，奖励归属规则（首次邀请者或随机）。假设仅首邀有效（需确认）。
7. **奖励规则**：
   - 完成测试即可获得“脆皮打工人”专属头像框（有效期15天）。
   - 评论区随机抽取50名用户，获得“终极养护礼包”。抽取是否前置条件（如必须发布动态或评论过？仅评论即参与？评论次数？）未明确。
8. **互动规则**：官方运营号带头评论，带动用户参与。用户评论允许玩梗语气，内容是否前置审核（猜测：不需要实时审核，允许展示但涉及违规可下架或删除）。
9. **话题区规则**：活动话题区内容为用户自行发布，平台是否保留审核权？是否支持用户删除自己发布的内容？未明确。
10. **导流入口规则**：活动页“专业养护通道”与圈子入口的展示逻辑（如用户完成测试后才展示，还是一开始就可见）待确认。

## 7. 数据字段与状态流转
- **核心对象：用户测试记录**
  - 字段：用户ID、测试ID、所选答案（10个选项）、总分、等级（轻度/中度/重度）、测试完成时间。
  - 状态流转：未开始 -> 答题中 -> 已完成 -> 已发布动态（可选）。
  - 状态约束：一旦“已完成”，不允许重新答题或修改答案？待确认。
- **核心对象：诊断书**
  - 字段：诊断书ID、用户ID、等级、结果文案、症状文案、医嘱文案、平台logo标识、生成时间。
  - 状态：生成成功 -> 已保存/已分享 -> 已发布动态（可选）。
- **核心对象：邀请记录**
  - 字段：老用户ID、新用户ID、邀请链接ID、邀请时间、新用户注册时间、新用户测试完成时间、新用户首次互动时间、奖励发放状态（已发放/未发放）。
  - 状态流转：已生成 -> 已点击 -> 已注册 -> 已完成测试 -> 已完成互动 -> 已发放奖励。
- **核心对象：活动奖品**
  - 头像框：关联用户ID，有效期15天（从获得时间开始）。
  - 终极养护礼包：关联用户ID，发放时间（抽奖后），使用状态（待确认）。
  - 脆皮养护礼包：关联老用户ID，发放条件满足后，使用状态（待确认）。
- **状态流转图（假设，需确认）**：用户注册 -> 测试完成 -> 诊断书生成 -> 动态发布 -> 邀请记录触发 -> 奖励发放。状态间流转依赖条件（如邀请记录需新用户完成所有行为才能触发奖励）。

## 8. 接口与依赖系统
- **依赖接口**：
  - *用户注册/登录接口*：供新用户注册，获取用户ID。
  - *测试题目获取接口*：从后端获取10道题目及选项（含文字和图片URL）。**图片URL替换为真实CDN地址，待接口文档提供**。
  - *答题提交接口*：用户提交全部答案，后端计算总分并返回等级/诊断书内容。返回字段需包含诊断书所有文案和图片URL。
  - *诊断书分享/保存接口*：触发分享时记录，或直接从生成接口返回可分享的图片URL。
  - *动态发布接口*：用户发布诊断书到话题区。需包含配文内容。
  - *话题区内容列表/评论接口*：获取动态列表、提交评论、删除评论、评论点赞等。
  - *邀请链接生成/校验接口*：老用户生成邀请码，新用户注册时校验是否通过邀请。
  - *奖励发放接口*：根据条件自动发放头像框/礼包。
  - *专业通道/圈子跳转接口*：活动页点击时携带用户ID跳转。
- **依赖系统**：
  - **用户系统**：用户注册、登录、信息管理。
  - **内容管理后台**：管理测试题、诊断书文案、奖品配置。能力推测：运营人员可配置题目、选项、分值。需后台支持修改活动内容。
  - **邀请系统**：生成邀请链接/海报、校验新用户来源、记录邀请数据。
  - **奖励系统**：奖品发放、有效期管理。
  - **推送/消息系统**：发送互动提醒、奖励领取通知（假设）。
  - **AI服务/逻辑**：计算总分、判断等级、生成诊断书文案。
- **埋点**：需统计活动参与人数、测试完成率、诊断书分享率、新用户注册数、有效邀请数、互动评论数、各入口点击率。依赖前端埋点与后端数据统计。

## 9. 测试范围建议
- **功能测试**：
  - 测试题展示与答题交互（10题全部可选，不可多选/跳题）。
  - 分值计算与等级划分（边界值：0, 9, 10, 15, 16, 25, 26, 40, 41 分对应的等级）。
  - 诊断书生成内容展示（三个等级文案、症状、医嘱、平台logo）。
  - 诊断书保存与分享功能（iOS/Android/微信分享兼容性）。
  - 用户发布动态与评论互动。
  - 邀请链接/海报生成、新用户注册校验、奖励触发。
  - 头像框有效期15天管理（是否到点失效）。
  - 评论抽奖活动（随机算法、50名抽取准确、重复抽取排重）。
- **兼容性测试**：
  - H5活动页在常见浏览器（Chrome、Safari、微信内置浏览器）的显示与交互。
  - 不同手机型号（主流Android/iOS）布局适配。
- **性能与负载测试**：
  - 模拟大量用户并发访问（尤其是活动上线初期）。
  - 诊断书生成与分享图片生成、动态发布接口的响应时间。
  - 邀请链接校验与奖励发放的高并发处理。
- **异常与边界测试**：
  - 网络中断后用户数据丢失/重试恢复。
  - 测试题未作答提交（提示）。
  - 同个用户多次测试/邀请自己/无效邀请行为（如未完成测试）。
  - 邀请链接重复使用或有效期。
  - 长时间无操作会话过期。
- **安全测试**：
  - 邀请链接防刷（避免批量注册产生大量无效邀请）。
  - 评论区防XSS/恶意内容。
  - 用户数据隐私保护（诊断书内容不可泄露给第三方）。
- **回归测试**：涉及用户系统、发帖/评论、邀请系统、头像框/礼包等已有功能的联动，建议针对会员系统、分享接口等执行回归。

## 10. 风险点与影响面
| 风险点 | 影响面 | 建议措施 |
|---|---|---|
| **时效性风险**：测试题/诊断语文案在活动期间传播过时，用户兴趣下降。 | 裂变和互动效果减弱，流量未导向核心功能。 | 预设可替换文案；预留后台动态修改测试题功能（能力）。 |
| **分数计算/等级划分错误**：AI计算逻辑有误或边界值（如15分=轻度，16分=中度，存在15-16临界）导致误判。 | 用户投诉，影响活动信任度。 | 重点测试两个相邻等级边界（14/15/16/25/26）以及总分异常值。 |
| **邀请系统刷量**：大量虚假注册/未完成测试骗取奖励。 | 奖励被滥用，真实用户奖励被稀释。 | 增加邀请有效性验证条件：必须完成测试+互动；AI可识别异常行为。 |
| **新用户判定不准确**：老用户开小号/同一设备多账号。 | 奖励发放不公，规则被绕过。 | 后端需记录设备ID+账号，启用风险策略（如同一设备下N个账号；待补充）。 |
| **后台配置能力不足**：运营无法动态修改测试文案/奖品配置。 | 活动弹性差，上线后无法调整。 | 上线前需确认后台管理界面是否支持活动配置（题目、文案、奖品等）。 |
| **图片/UI缺失**：诊断书生成依赖于图片样式（含logo），若设计未按时交付。 | 分享页卡顿、无法展示，影响功能体验。 | 提前预设占位图，预留图片替换接口或CDN更新。 |
| **交互规则不明确**：“求会诊”“求药方”的发布内容是否强制（必须配文？）是否允许修改？ | 用户体验不统一，审核困难。 | 明确规则：允许用户自定配文，但给出建议选项。 |
| **分享到微信等第三方平台兼容性问题** | 分享卡片无法正常显示，影响裂变传播。 | 针对微信、QQ、微博等平台测试分享SDK。 |

## 11. 待确认问题
1. **图片内容细节缺失**：输入材料中频繁出现图片引用（如测试题选项图片、诊断书示例图、症状对应图片等）。请人工提供这些图片的详细说明（位置、文案、交互逻辑），我不应猜测图片内容。
2. **答题提交接口返回结构**：用户提交答案后，接口返回的JSON具体结构是什么？是否包含诊断语文案、图片URL、甚至预先生成的分享图URL？
3. **新用户判定标准**：实现“新用户首次注册”的判定是基于设备ID+手机号重复注册检测，还是仅手机号？结合邀请逻辑，老用户是否能用同一设备多账号刷单？
4. **同一用户多次参与测试规则**：同一账号是否允许重复答题？若允许，诊断书文案是否更新？发布动态时是覆盖还是新增？
5. **邀请奖励上限**：一个老用户可邀请多个新用户，累计邀请奖励有无上限？若有多级奖励（比如邀请2人得A，3人得B），需确认。
6. **终极养护礼包抽取机制**：“评论区随机抽取50名” - 抽取算法是实时（从所有评论中）还是积累后一次性抽取？是否限制用户从多个评论后被重复抽取？
7. **头像框有效期计算**：“有效期15天”是从用户获得头像框那一刻开始算，还是从下一次登录/使用时开始算？
8. **用户发布内容审核**：活动话题区是否需要内容审核机制？如用户发布的不合规内容（广告、人身攻击）如何处理？是否采用关键词拦截？
9. **邀请链接/海报有效期**：老用户生成的专属链接或海报是否有过期时间？
