# 需求分析草稿：脆皮打工人养护中心 - 5月活动玩法

## 1. 需求背景与目标

**需求背景**：基于5月活动玩法PRD，通过H5互动测试、社交裂变和社区导流，提升用户活跃度和平台核心功能（职场圈、治愈圈、专业咨询板块）的长期留存，形成“活动引流→长期留存”的闭环。

**业务目标**：
- 用户参与脆皮测试并生成专属电子诊断书，可保存/分享，强化传播。
- 通过邀请好友玩法带动新用户注册与参与。
- 将活动流量导流至平台职场圈、治愈圈、专业咨询板块，降低活动后流失率。

## 2. 业务范围

| 范围类型 | 具体内容 |
|:---|:---|
| **范围内** | 脆皮测试（10道选择题）、AI自动评分、诊断书生成与分享、邀请好友裂变、活动话题区互动（发帖/评论/会诊）、奖励发放（头像框、养护礼包）、导流入口（专业圈、话题区、主理人） |
| **范围外** | 奖励的实际物流/发放系统（假设依赖现有平台奖励/库存系统）、H5活动页面的具体UI/UX设计（需依赖设计图）、主理人线上茶话会的实际落地（仅提供入口/联动，未提具体系统实现） |
| **待确认/假设** | 图片内容未分析，生成诊断书时具体的图片素材、布局、按钮文案、交互细节均依赖设计图，本需求分析仅基于文本。 |

## 4. 主流程拆解

**主流程1：用户参与脆皮测试**
1. 用户通过H5活动页/分享链接进入。
2. 用户依次回答10道选择题（每题ABCD四选项）。
3. AI自动计算总分：A=4, B=3, C=2, D=1。
4. 根据总分判定层级（10-15=轻度，16-25=中度，26-40=重度）。
5. 生成对应等级的专属电子诊断书（含诊断结果、症状、医嘱、玩梗文案）。
6. 用户可保存/分享诊断书（生成带平台logo的图片）。

**主流程2：用户互动**
1. 用户生成诊断书后，可在活动话题区发布内容（配文如“求会诊”“求药方”）。
2. 其他用户/运营号在话题区进行评论（“会诊”）。
3. 官方运营号带头评论，引导用户互动。

**主流程3：邀请好友裂变**
1. 老用户生成专属邀请链接/海报。
2. 新用户通过链接/海报注册APP并参与活动。
3. 新用户完成脆皮测试并发布1条互动内容。
4. 系统判定为有效邀请，老用户获得奖励（脆皮养护礼包/免费专业养护通道）。

**主流程4：导流**
1. 活动页设置“专业养护通道”入口，吸引用户关注专业圈（心理、养生、职场）主理人。
2. 活动话题区置顶“职场摸鱼圈”“治愈放松圈”“脆皮养护交流圈”入口，引导用户进入圈子。
3. 平台邀请主理人发起“线上茶话会”，通过活动页/圈子入口吸引用户参与。

**主流程5：奖励发放**
1. 用户完成测试→立即发放“脆皮打工人”专属头像框（有效期15天）。
2. 互动评论→评论区随机抽取50名用户→发放终极养护礼包。

## 5. 分支流程与异常流程

| 流程类型 | 场景 | 描述 |
|:---|:---|:---|
| **分支** | 测试结果层级判定 | AI自动计算总分，按不同分数段输出专属诊断书。 |
| **分支** | 邀请链接/海报生成失败 | 系统需有重试机制或返回错误提示。 |
| **分支** | 新用户未完成测试或未发布互动内容 | 仅注册不视为有效邀请，老用户未获得奖励。 |
| **异常** | 用户网络中断 | 测试题目保存进度(假设)或重新作答(待确认)。 |
| **异常** | AI评分系统故障 | 返回失败提示，用户无法生成诊断书。 |
| **异常** | 邀请链接/海报过期或失效 | 新用户点击后提示“邀请已失效”。 |
| **异常** | 奖励发放失败（头像框/养护礼包） | 需有重试机制或补发策略。 |
| **异常** | 用户重复提交测试 | 是否限制同一用户只能参与一次？（待确认） |
| **异常** | 话题区评论包含违规内容 | 是否触发屏蔽/审核机制？（待确认，假设无） |

## 10. 风险点与影响面

| 风险编号 | 风险描述 | 影响面 | 优先级 |
|:---|:---|:---|:---|
| Risk-1 | AI评分服务不稳定或返回错误，用户无法完成测试。 | 核心玩法崩溃，用户体验受损，活动目标无法达成。 | P0 |
| Risk-2 | 高并发下，邀请/裂变系统或奖励系统接口超时或数据不一致。 | 邀请记录、奖励发放异常，引发客诉。 | P0 |
| Risk-3 | 头像框有效期15天逻辑未正确实现，提前失效或永久有效。 | 用户感知混乱、资产异常。 | P1 |
| Risk-4 | 诊断书图片生成依赖的图片素材缺失或格式不兼容。 | 诊断书生成失败或显示异常。 | P1 |
| Risk-5 | 活动话题区未实现“会诊”玩法的特殊标识/逻辑，导致普通评论与官方评论区分混乱。 | 互动玩法体验下降。 | P1 |
| Risk-6 | 邀请链接/海报被恶意刷量或分享至非法渠道。 | 带来垃圾注册、刷奖励风险，影响平台安全。 | P2 |
| Risk-7 | 测试规则（10题固定）不支持动态调整，后续需修改时成本高。 | 应对未来运营策略变化的灵活性差。 | P2 |
| Risk-8 | 用户设备（如低端手机/旧浏览器）兼容性差，导致H5页面布局错乱或图片无法生成。 | 降低活动参与率，用户体验差。 | P2 |
| Risk-9 | (待确认)未明确同一用户是否可重复测试，可能导致刷分或多次生成诊断书。 | 逻辑混乱，浪费系统资源，数据统计不准确。 | P2 (需确认) |
| Risk-10 | 失败/重复请求的幂等性未实现，可能导致奖励重复发放或数据错乱。 | 奖励事件频繁，系统负担大。 | P1 |

## 11. 待确认问题

**问题1**：用户是否可以多次参与同一套测试题？如果可以，是否保留历史测试记录或更新最新诊断书？（目前PRD未明确，建议确认后增加限制：每人仅限一次或每日/每周一次。）

**问题2**：邀请链接/海报的有效期是多久？活动期间一直有效还是有过期时间规定？新用户注册后多久内完成测试+发布才算有效邀请？（PRD未明确，需运营/产品确认）

**问题3**：活动话题区的“会诊”评论是否需要特殊标识（如标签、样式）以区别于普通评论？官方运营号如何定位和参与？（PRD仅描述玩法和效果，未明确技术实现细节。）

**问题4**：图片内容未分析。PRD中多次引用图片（如测试题图片、诊断书样式、症状对应图片、活动宣传页设计图等），本需求分析基于纯文本生成，无法分析图片中的字段、按钮、布局、交互逻辑等。建议人工接入设计图后补充分析。

**问题5**：互动奖“评论区随机抽取50名”，随机抽取的具体算法和可见性如何？是否为所有评论者？是否排除马甲/水军账号？（需确认抽取逻辑的公平性和技术实现。）

**问题6**：奖励“终极养护礼包”和“免费专业养护通道”的具体内容是什么？是否依赖平台现有奖励系统/库存系统？发放是否有库存/限额限制？（需运营明确奖励具体内容及库存。）
