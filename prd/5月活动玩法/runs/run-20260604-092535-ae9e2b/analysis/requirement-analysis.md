# 需求分析草稿：5月活动玩法 - 脆皮打工人养护中心

## 1. 需求背景与目标

- **背景**：基于 PRD `prd/5月活动玩法/input/requirement.md`，活动主题为“脆皮打工人养护中心”，slogan为“拒绝内卷，在线会诊”。旨在通过玩梗的测试、诊断书生成、社区互动和裂变机制，提升用户活跃度、参与感和传播效果。
- **目标**：
    1.  **用户参与**：用户通过完成10道脆皮测试题，获得个性化诊断报告。
    2.  **社交传播**：鼓励用户生成专属诊断书并分享到社交平台，实现裂变。
    3.  **社区互动**：在活动话题区引导用户发布“求会诊”“求药方”内容，并通过官方运营带动社区活跃。
    4.  **用户拉新**：通过“邀请好友，双向有礼”的裂变玩法，吸引新用户注册与参与。
    5.  **平台导流**：将活动流量导流至平台的核心功能模块（职场圈、治愈圈、专业咨询等），提升用户长期留存。

## 2. 业务范围

- **范围内**：
    - H5/App内脆皮测试题目展示与答题交互。
    - AI自动评分，生成轻、中、重度3类格式化的诊断书（附带玩梗文案）。
    - 诊断书生成结果为可保存、可分享的图片（含平台logo）。
    - 活动话题区的发布“求会诊”“求药方”内容功能。
    - 用户评论互动（“会诊”）。
    - 邀请好友裂变功能（生成邀请链接/海报，绑定新老用户关系）。
    - 活动奖励发放（参与奖：专属头像框；互动奖：终极养护礼包）。
    - 活动页面到平台其他核心模块的导流入口。
    - 官方运营账号在话题区的互动功能。
- **范围外（待确认）**：
    - “给室友开个方子”和“定时红包活动”这两个子玩法的具体实现（PRD仅提及，未提供细节，本次分析仅分析核心玩法）。
    - 诊断书生成的图片模板具体设计（仅提供文案，未提供模板）。
    - 邀请裂变功能的完整实现（如邀请记录、奖励到账通知）。
    - 专业咨询/主理人板块的深度集成和具体交互。
    - 后台管理端的具体操作界面和配置功能。

## 3. 角色与权限

- **用户角色与权限（假设）**：
    | 角色 | 权限 |
    |---|---|
    | **游客（未登录）** | 可浏览活动页面、查看测试题目、查看活动规则。**假设**：不可参与答题、生成诊断书、发布互动内容、进行邀请裂变。 |
    | **新用户** | 可参与脆皮测试、获取诊断书、获取参与奖、受邀时完成注册。 |
    | **老用户（已注册）** | 可参与所有活动、生成邀请链接/海报、邀请新用户、获得邀请奖励、在话题区发布/评论/分享。 |
    | **官方运营账号** | 可在话题区带头评论、发布玩梗内容。**假设**：具有特殊标识或特权。 |
- **后台管理员角色（假设）**：
    - **活动配置管理员**：配置活动时间、奖励、测试题目、评分规则等。
    - **内容管理**：管理话题区内容（删除违规帖、置顶帖）。
    - **运营管理**：选择互动奖中奖用户、发放奖励。
    - **数据查看**：查看活动参与数据、裂变数据、用户行为数据。
- **权限差异**：
    - **游客**和**登录用户**的权限差异是核心，游客无法完成核心玩法的闭环。
    - 不同角色在社区互动中的可见性、操作权限需要明确。

## 4. 主流程拆解

- **主流程**：用户访问活动页面 → 点击“开始测试” → 依次回答10道选择题（每题4个选项）→ 提交 → 系统AI自动评分 → 生成专属诊断书图片（含文案） → 用户可保存、分享、发布到话题区。
- **裂变主流程（邀请）**：老用户在活动页面点击“邀请好友” → 系统生成专属邀请链接/海报 → 老用户通过社交平台分享 → 新用户点击链接/扫码 → 注册/登录并参与活动（完成测试+发布1条互动内容）→ 系统判定邀请成功 → 老用户获得奖励。
- **互动主流程（会诊）**：用户发布“求会诊”内容 → 其他用户或官方账号在评论区回复“会诊”/“开药方”。

## 5. 分支流程与异常流程

- **分支流程**：
    - 用户未完成所有题目时中途退出。
    - 用户选择D选项（如“活力满满（异类）”），可能中断正常计分逻辑（假设该类选项不计入严重程度计分，或生成特殊诊断书）。
    - 用户生成诊断书后，选择“发布到话题区”而非保存/分享。
    - 用户通过邀请链接进入活动但未完成全部任务（注册+测试+发布）就退出。
- **异常流程**：
    - **网络异常**：
        - 答题提交时网络中断，应提示用户重试，已答题目应不被清空。
        - 邀请链接加载失败、海报生成失败。
        - 发布内容失败。
    - **数据异常**：
        - 评分规则计算失败（如选项A的值被错误计算）。
        - 同一个邀请被多个新用户重复绑定，或一个老用户奖励被多次领取。
        - 诊断书生成时，图片文案或占位符未能正确替换。
        - 活动奖励（头像框、礼包）发放失败或重复发放。
    - **用户操作异常**：
        - 用户在答题中快速点击提交。
        - 用户使用外挂或脚本批量答题、刷邀请。
        - 用户在话题区发布违规内容。
        - 用户（新用户）在注册流程中退出，导致邀请关系无法绑定。
    - **并发异常**：
        - 大量用户同时进入活动页面，导致页面加载慢或服务挂起。
        - 大量用户同时生成诊断书，导致图片服务负载过高。
        - 大量邀请同时进行，导致奖励计算错误。
    - **依赖异常**（假设）：
        - 专业咨询/主理人板块接口不稳定，导致导流链接失效。
        - 图片生成服务（如第三方服务）超时或返回错误。

## 7. 数据字段与状态流转

- **核心业务对象**：
    - **活动**：活动ID、活动名称（“脆皮打工人养护中心”）、活动开始/结束时间（**待确认**）、活动状态（未开始、进行中、已结束）。
    - **测试题目**：题目ID、题目内容、选项（A/B/C/D，每个选项对应分数值）、展示顺序。
    - **用户答题记录**：用户ID、题目ID、所选选项、题目得分。
    - **诊断结果**：用户ID、总分、脆皮等级、诊断书ID、生成时间。
    - **诊断书图片**：图片URL、用户ID、诊断书ID。
    - **邀请记录**：邀请ID、发起用户（老用户）、受邀用户（新用户）、邀请时间、状态（待完成、已完成、已失效）。
    - **奖励记录**：用户ID、奖励类型（头像框、礼包）、奖励ID、发放时间、有效期。
    - **互动内容**：内容ID、发布者ID、内容文本、发布时间、类型（“求会诊”“求药方”）。
- **状态流转**：
    - **邀请记录**：`待完成`（新用户注册但未完成指定任务）→ `已完成`（新用户完成测试+发布互动）→ `已失效`（活动结束或邀请过期）。
    - **用户参加活动**：`未开始`（浏览页面）→ `答题中`（已开始答题）→ `已提交`（完成提交）→ `已生成诊断书` → `已发布/已分享`。

## 8. 接口与依赖系统

- **目前无API文档**（PRD中 `input/api.md` 无内容）。以下接口为基于需求的推测，需要在后续确认：
    - **测试答题接口**：`POST /api/quiz/submit` - 提交用户答题数据（用户ID、答案列表）。返回总分、脆皮等级和诊断书信息。
    - **诊断书生成接口**：`POST /api/diagnosis/generate` - 根据用户ID和脆皮等级生成诊断书图片。返回图片URL。
    - **邀请接口**：
        - `POST /api/invite/generate` - 老用户生成邀请链接/海报。返回链接/海报数据。
        - `GET /api/invite/land?code=xxx` - 新用户通过邀请链接访问，校验邀请码。
        - `POST /api/invite/reward` - 当新用户完成任务后，通知后端给老用户发放奖励。
    - **社区互动接口**：
        - `POST /api/content/publish` - 发布“求会诊”/“求药方”内容。
        - `GET /api/content/list` - 获取话题区内容列表。
        - `POST /api/comment/reply` - 用户评论“会诊”。
    - **奖励发放接口**：`POST /api/reward/grant` - 发放头像框、礼包等奖励。
    - **用户信息接口**：`GET /api/user/info` - 获取用户登录状态和基本信息。
- **依赖系统**：
    - **用户系统**：用于判断用户登录状态、用户信息、新用户注册。
    - **奖励系统**：用于发放和领取奖励。
    - **图片生成服务**：用于生成诊断书图片。
    - **消息通知系统**：用于通知用户奖励领取、邀请成功等。
    - **第三方分享平台**：微信、微博等，用于分享诊断书和邀请链接。
    - **官方运营账号系统**：用于官方账号的社区互动。

## 9. 测试范围建议

- **功能测试**：
    - **测试题目模块**：10道题目的展示、选项选择、提交逻辑、选项分数计算。
    - **评分与诊断书模块**：总分计算、脆皮等级判定、诊断书文案生成、图片生成、保存、分享、发布功能。
    - **邀请裂变模块**：老用户生成邀请链接/海报、新用户通过链接注册/参与、邀请状态跟踪、奖励发放。
    - **社区互动模块**：话题区发布“求会诊”/“求药方”、评论“会诊”、官方账号互动。
    - **奖励模块**：参与奖和互动奖的发放时机、逻辑、收货信息（如需要）。
    - **导流模块**：各导流入口（专业养护通道、圈子等）的跳转正确性。
- **流程测试（端到端）**：覆盖主流程、分支流程、异常流程。
- **兼容性测试**：H5页面在不同手机尺寸、操作系统（iOS、Android）、不同浏览器上的兼容性。
- **性能测试**：重点场景：大量用户同时答题、生成诊断书（图片服务）、大量邀请并发。
- **安全性测试**：
    - 防止刷奖励（头像框重复领取、礼包多次抽取）。
    - 防止外挂脚本批量答题。
    - 防止利用邀请链接进行恶意注册。
- **数据一致性测试**：
    - 用户答题记录、诊断结果、邀请记录、奖励记录数据一致。
    - 诊断书文案与脆皮等级的匹配关系。
- **埋点与日志测试**：验证关键用户行为（点击测试、完成答题、生成诊断书、分享、邀请、发布内容、点击导流）的埋点正确上报。
