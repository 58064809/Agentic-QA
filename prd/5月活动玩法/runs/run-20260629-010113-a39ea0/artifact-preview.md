## 1. 需求背景与目标

**背景**：PRD 为 5 月活动玩法中的“脆皮打工人养护中心”H5 活动，目标用户为“脆皮打工人”，通过玩梗测试引发共鸣，降低参与门槛，带动话题互动、裂变拉新和平台核心功能导流。

**目标**：
- 用户完成 10 道脆皮测试题，AI 自动评分并生成专属电子诊断书（含等级、症状、医嘱、玩梗文案、图片），可保存分享，强化传播。
- 用户可在活动话题区发布诊断书并互动“会诊”，其他用户评论开药方，官方运营带头带动参与。
- 老用户邀请新用户参与活动（新用户需完成测试+发布1条互动内容），老用户获得奖励，实现裂变拉新。
- 通过“专业养护通道”“圈子导流入口”“主理人联动”将活动流量导流至平台职场圈、治愈圈、专业咨询板块，提升长期留存。

## 2. 业务范围

### 范围内

- 导流功能（专业养护通道、圈子入口、主理人线上茶话会入口）

## 4. 主流程拆解

### 4.1 用户参与脆皮测试并生成诊断书
1. 用户进入 H5 活动页（可通过链接、二维码、APP 内入口）
2. 浏览活动页介绍及 Slogan“拒绝内卷，在线会诊”
3. 开始答题：依次回答 10 道单选题，每题 4 个选项
4. 提交所有答案（需确认是否允许中途退出/暂存）
5. 系统 AI 自动计算总分（A=4, B=3, C=2, D=1），并划分等级：
   - 10-15 分：轻度脆皮
   - 16-25 分：中度脆皮
   - 26-40 分：重度脆皮
6. 生成诊断书：包含诊断结果、症状、医嘱，并合成带平台 logo 的图片
7. 用户可选择保存到本地、分享给好友、或直接发布到活动话题区

### 4.2 用户发布诊断书并参与互动
1. 用户在诊断书生成后，点击“发布到活动话题区”
2. 配文（可输入自定义文本，或使用默认“求会诊”“求药方”）
3. 发布成功，其他用户可看到该帖
4. 其他用户可在评论区“会诊”（文字评论，可带玩梗文案）
5. 官方运营号带头评论，带动氛围

### 4.3 裂变邀请（带动新用户）
1. 老用户（已登录且在活动页面）生成专属邀请链接/海报
2. 分享至外部渠道（微信、朋友圈等，待确认支持哪些渠道）
3. 新用户通过链接/海报二维码注册 APP（需确认是否必须通过本链接注册）
4. 新用户完成脆皮测试+发布 1 条互动内容
5. 系统判定为有效邀请，老用户获得奖励（“脆皮养护礼包”或“一次免费专业养护通道活动”）
6. 新用户同时可获得参与奖（头像框）

### 4.4 导流至平台核心功能
1. 活动页设置“专业养护通道”入口，点击进入可看到主理人发布的专业建议
2. 活动话题区置顶“职场摸鱼圈”“治愈放松圈”“脆皮养护交流圈”入口
3. 用户点击进入对应圈子，长期沉淀社交关系
4. 主理人发起线上茶话会（社群语音/文字），用户报名参与

### 4.5 奖励发放
1. 完成测试：系统自动发放专属头像框（有效期 15 天）
2. 互动奖：在话题区评论区随机抽 50 名用户，每人一份“终极养护礼包”（具体内容待确认）
3. 邀请奖：老用户根据有效邀请数量获得奖励（发放规则待确认）

## 5. 分支流程与异常流程

| 分支/异常场景 | 处理方式 |
|---------------|----------|
| 用户答题过程中退出 | 假设：已答题进度不保存，下次进入需重新答（待确认） |
| 用户未完成全部 10 题即提交 | 系统应提示“请完成所有题目”，或不允许提交？待确认 |
| 总分未在有效范围内（如 0 分、低于 10 分） | 由于每题至少 1 分，10 题最低 10 分，理论上不会出现，但边界测试需覆盖 |
| 诊断书图片生成失败 | 系统应返回错误提示，允许用户重试；文本版诊断书仍可展示（待确认） |
| 用户保存诊断书时相册权限未授权 | 引导用户授权，或提供其他保存方式（如长按保存） |
| 发帖时内容包含敏感词 | 按平台现有内容审核机制处理；是否允许发空内容？ |
| 新用户未完成测试或未发布互动 | 不计入有效邀请；老用户无法获得该邀请奖励 |
| 邀请链接被恶意刷量 | 需防刷机制：同一设备/ID 注册关联限制；邀请奖励防重复领 |
| 头像框发放失败 | 补偿机制？需确认是否有重试逻辑 |
| 互动奖抽奖：用户已删除评论 | 是否仍参与抽奖？需确认抽奖逻辑是基于评论 ID 还是用户 ID |
| 用户未登录直接访问活动页 | 可浏览但无法答题？（假设：需登录才能开始答题，避免匿名刷数据） |
| 多人同时邀请同一个人 | 以首次注册链接为准？待确认 |
| 活动结束后的奖励领取 | 奖励需在活动期间领取？头像框有效期从领取时算起？ |

## 11. 待确认问题

1. **用户登录要求**：用户是否需要登录才能开始答题？生成诊断书、保存图片、发布话题、参与邀请等功能是否强制要求登录？如果未登录，是否允许浏览题目但不记录分数？
2. **答题完整性**：如果用户只回答部分题目就提交，系统如何处理？是否允许跳过某题？提交时是否校验所有题目已答？
3. **互动内容定义**：PRD 要求新用户“发布1条互动内容”才算有效邀请，互动内容具体指什么？是发布诊断书到话题区，还是任意评论都算？是否对内容字数有要求？
4. **诊断书图片具体内容**：由于图片引用已省略，无法分析诊断书图片的布局、包含的文字字段、平台 logo 位置和样式。请设计师提供设计稿，以便评估图片合成逻辑的测试点。
5. **邀请链接/海报生成**：支持哪些分享渠道（微信、朋友圈、微博、QQ等）？二维码内容是什么？是否包含参数追踪来源？
6. **奖励明细**：
   - “脆皮养护礼包”具体包含什么？虚拟物品还是实物？发放方式？
   - “一次免费专业养护通道活动”的具体形式是什么（咨询券、优惠码等）？
   - “终极养护礼包”包含哪些内容？与“脆皮养护礼包”有何区别？
7. **头像框有效期**：15 天有效期从何时开始计算？从领取时刻？从活动结束时间？是否支持续期？
8. **导流入口跳转规则**：点击“专业养护通道”后，是跳转到 APP 内某个页面（如职场圈）还是 H5 页面？是否需要在当前活动页打开新标签页？是否涉及页面区分已登录/未登录用户？
9. **主理人联动**：线上茶话会是使用第三方平台（如微信社群）还是 APP 自有功能？如果使用自有功能，音频/文字聊天功能是否已存在？是否需要新开发？
10. **题目展示顺序**：10 道题是否固定顺序？答完一题后是自动跳转下一题还是手动点击？每题是否有倒计时？
11. **并发与性能要求**：是否需要对高并发（如瞬间大量用户参与）做性能测试？是否有预估并发量？
12. **运营后台上传题目/文案**：是否提供后台管理功能，允许运营修改题目、选项、分值、诊断书文案？当前需求未提及，但如果不支持，下次活动需重新发版。
