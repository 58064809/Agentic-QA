## 1. 需求背景与目标

- **业务目标**: 通过“脆皮打工人养护中心”H5活动，以玩梗、互动和社交裂变的方式，在5月活动期间提升用户活跃度、拉新用户，并最终将活动流量导流至平台核心圈子（职场圈、治愈圈、专业咨询板块）和主理人生态，实现用户留存。
- **用户价值**: 为用户提供一种趣味性的自我诊断和情绪宣泄方式，通过分享诊断书和社区互动获得认同感和娱乐感。新老用户可通过邀请好友获得奖励。
- **核心指标**:
    - 用户参与率（完成脆皮测试的用户数）
    - 诊断书生成与分享率
    - 活动新增用户数（通过裂变）
    - 导流到各圈子的转化率
    - 互动评论数

## 2. 业务范围

- **范围内**:
    1.  H5活动首页：展示活动主题“脆皮打工人养护中心”，带玩梗风格的Slogan“拒绝内卷，在线会诊”。
    2.  问卷与AI评分：用户完成10道选择题后，AI根据用户选择自动计算得分并划分脆皮等级（轻度、中度、重度）。
    3.  诊断书生成：根据分数等级生成对应的玩梗诊断结果文案和症状描述，并可生成带平台logo的图片供用户保存和分享。
    4.  社交分享与裂变：
        - 用户可生成专属邀请链接/海报，邀请新用户注册参与。
        - 被邀请的新用户完成测试并至少发布1条互动内容，视为有效邀请。
    5.  社区互动：
        - 用户可发布诊断书到活动话题区并自动生成配文“求会诊”“求药方”。
        - 其他用户可在评论区回复“会诊”并提供“药方”。
        - 官方运营号带头用玩梗语气回复评论。
    6.  导流机制：
        - 活动页设有“专业养护通道”，可跳转至专业圈主理人的内容或咨询入口。
        - 活动话题区置顶“职场摸鱼圈”“治愈放松圈”“脆皮养护交流圈”的入口。
    7.  奖励发放：
        - 完成测试即可获得“脆皮打工人”专属头像框（APP内使用，有效期15天）。
        - 评论区随机抽取50名获得“终极养护礼包”。

- **范围外** (假设/待确认):
    1.  其他类型的H5互动游戏（如“给室友开个方子”等，仅提及活动方案链接，未提供具体需求）。
    2.  定时红包活动（仅提及活动方案链接，未提供具体需求）。
    3.  实际积分商城或优惠券的兑换流程。
    4.  专业主理人进行的线上“茶话会”活动执行细节和平台底层功能。

## 3. 角色与权限

- **用户角色**:
    - **老用户**: 有邀请权限（生成链接/海报），获得邀请新用户奖励。可参与测试、分享、评论。
    - **新用户**: 通过链接/海报注册，参与活动并发布互动内容。无发起邀请权限。
    - **其他用户**: 平台所有注册用户（非直接参与测试，但在话题区可见活动内容）。
- **管理后台**:
    - **官方运营账号**: 在评论区进行“会诊”引导，带动话题。
    - **入驻主理人**: 在“专业养护通道”发布建议内容。需后台内容管理系统支持。 (假设：主理人角色已有，需确认该平台是否已有该后台权限)
- **系统服务**:
    - AI评分引擎：根据用户答案和计分规则自动计算。
    - 图片生成服务：根据诊断结果和平台logo生成诊断书图片。
    - 邀请链接/海报服务：生成并识别邀请关系。

## 4. 主流程拆解

1.  **用户进入活动**：
    用户通过H5入口/海报扫码/分享链接进入“脆皮打工人养护中心”活动页。
2.  **完成脆皮测试**：
    用户在活动页依次作答10道关于职场状态和习惯的选择题。
3.  **获得AI评分与诊断**：
    提交测试后，系统根据预设规则计算总分，判断用户属于“轻度/中度/重度脆皮”，并展示对应的“诊断结果”、“症状”和“医嘱”文案。
4.  **生成并分享诊断书**：
    用户可以操作：1) 将诊断结果文案保存/分享；2) 生成带平台logo的诊断书图片，可保存至本地或通过社交渠道分享。
5.  **社区互动（发布与评论）**：
    用户将诊断书发布到活动话题区，自动生成“求会诊”“求药方”文案。其他用户可进入话题区，查看诊断报告并评论“会诊”和“药方”。
6.  **邀请好友与裂变**：
    老用户（已参与测试）生成专属邀请链接/海报，通过社交渠道分享给新用户。新用户通过特定渠道注册，完成测试并首次发布互动内容，则邀请流程生效。
7.  **导流至圈子**：
    用户可通过活动页面链接/按钮跳转至“职场摸鱼圈”、“治愈放松圈”、“专业养护通道”等。

## 5. 分支流程与异常流程

| 场景 | 流程描述 |
|---|---|
| **用户未提交全部10题** | 提交时判断是否所有题目已作答。若未答全，给出提示并要求全部完成。 |
| **AI评分系统故障** | 用户提交问卷后，AI无法正常返回评分。需显示友好提示（如“正在诊断中，请稍候”），或降级为默认结果/错误页。 |
| **图片生成服务异常** | 生成诊断书图片时，服务超时或报错。前端应提示用户，并提供纯文案分享作为降级方案。 |
| **邀请链接/海报失效** | 用户点击的邀请链接已过期或无效。系统应提示“邀请已失效”，并引导用户正常参加活动。 |
| **新用户非有效邀请** | 新用户注册后，仅答题但未在话题区发布1条互动内容。则该用户不计入老用户的“有效邀请”。 |
| **重复邀请** | 相同的新用户通过不同老用户链接注册。邀请关系如何判定？需确认是首次点击绑定还是最后一次注册绑定。 |
| **领取奖励失败** | 用户判断获得头像框后，发放API返回失败。应有重试机制或提示用户稍后查看。 |
| **专业养护通道跳转失败** | 点击导流入口，链接出错或目标圈子不存在。需有兜底页面，避免死链。 |

## 7. 数据字段与状态流转

- **核心数据实体:**
    1.  **用户测试问卷**: `user_id, question_id, answer (A/B/C/D), total_score, brittle_level`
    2.  **诊断书**: `user_id, level, diagnosis_text, diagnose_pic_url, created_time`
    3.  **好友邀请**: `inviter_id, invitee_id, invite_channel, invite_time, is_effective (true/false)`
    4.  **奖励记录**: `user_id, reward_type (头像框/礼包), status (待发放/已发放/已失效), start_time, end_time (头像框)`

- **脆皮测试状态流转**:
    `未开始` -> `答题中` -> `等待评分` -> `已出诊(轻度/中度/重度)`
    异常: `等待评分` -> `评分失败` -> `显示错误/降级`

- **诊断书分享状态流转**:
    `已出诊` -> `用户点击生成/分享` -> `分享成功/失败`

## 8. 接口与依赖系统

- **核心接口** (推断，需确认API文档):
    - `POST /api/activity/test/submit`: 提交用户十个选择题答案。
    - `GET /api/activity/test/result`: 获取用户诊断结果、等级。
    - `POST /api/activity/diagnosis/upload-bg`: 上传用户诊断文案+图片模板，生成分享图片。
    - `GET /api/activity/invite/generate-link`: 为用户生成个性化邀请链接/海报。
    - `POST /api/activity/invite/bind`: 记录新用户通过某个邀请链接注册。
    - `POST /api/activity/comment`: 用户提交评论。
    - `GET /api/activity/comment`: 获取评论区内容。
    - `POST /api/activity/reward/claim`: 用户领取头像框/礼包。

- **依赖系统**:
    - **用户注册/登录系统**: 用于注册与身份绑定。
    - **AI评分引擎**: 根据规则返回用户分数和等级。
    - **图片生成/处理服务**: 生成包含文案+logo的诊断书图片。
    - **社群内容服务**: 支撑话题区发帖、评论、置顶。
    - **消息推送系统**: 用于用户获得互动回复或奖励的通知。

## 9. 测试范围建议

- **功能测试**:
    - Android, iOS的H5内置浏览器展示效果与交互测试。
    - 10道选择题的正确点击与选中状态显示。
    - 不同答案组合的AI评分结果（边界值：10, 15, 16, 25, 26, 40分）。
    - 诊断书不同等级的文案、图片、logo正确展示。
    - 头像框领取后是否在APP内正确展示，有效期开始、到期和过期后是否失效。
    - 邀请链接/海报的生成与分享功能、新用户跳转到注册页功能。
    - 邀请关系的有效判定（新用户答题+评论）测试。
    - 评论区点赞、回复、排序、随机抽奖算法正确性。
    - 导流至各圈子的链接正确性和跳转无异常。
- **交互/UI测试**:
    - 诊断书图片的保存到本地、分享至微信/朋友圈功能。
    - 活动页面在不同分辨率下的响应式布局和美观性。
    - 各个按钮、卡片、弹窗的点击热区和动画效果。
    - 图片内容未分析，需人工确认UI和前端交互是否符合设计图。
- **数据与规则验证**:
    - 邀请关系绑定逻辑（重复邀请、并发生成链接等边界情况）。
    - 奖励（头像框）到期自动失效的规则验证。
    - 话题区自动配文和展示逻辑。
- **兼容性测试**:
    - 主流H5浏览器：微信内置浏览器、QQ浏览器、Chrome、Safari。
- **性能测试**:
    - 高并发下提交测试、生成图片、生成邀请链接等操作的响应时间与稳定性。
- **安全测试**:
    - AI评分结果的篡改验证（防止刷分）。
    - 邀请链接的参数防篡改和防伪造。
    - 用户奖励领取接口的幂等性验证。

## 10. 风险点与影响面

| 风险点 | 影响面 |
|---|---|
| **诊断书图片生成服务不稳定** | 用户无法生成和分享诊断书，影响分享裂变的传播效果和活动数据。 |
| **邀请链接的并发/重复绑定问题** | 可能导致邀请关系错乱，老用户无法获得奖励，引起用户投诉。 |
| **AI评分规则或裁判逻辑BUG** | 用户等级判定错误，导致诊断书文不对题，影响活动趣味性和品牌口碑。 |
| **头像框发放/到期逻辑BUG** | 用户未收到奖励或奖励失效时间错误，影响用户体验。 |
| **接口文档未提供** | 无法进行精确的接口测试，前端与后端的配合可能存在大量未预见的异常场景。 |
| **图片内容未分析** | 无法校验UI交互细节、文案是否匹配、活动玩法是否完整。所有UI层面的测试都依赖人工确认。 |
| **评论区随机抽奖算法未定义** | 无法设计有效的白盒测试用例，只能从黑盒观察结果。 |
| **历史知识/缺陷未提供** | 无法复用之前的相似案例和已知问题，需要从零分析和构造测试数据。 |

## 11. 待确认问题

1.  **[诊断书分享]** PRD中提到用户生成诊断书后可发布到活动话题区。当用户将诊断书分享到活动话题区时，是仅分享文案、仅分享图片，还是自动同时分享“文案+图片”？发布到社区后，其他用户看到的互动格式是什么？
2.  **[AI评分接口]** 核心的AI评分接口`/api/activity/test/submit`和`/api/activity/test/result`的详细请求/响应参数是什么？如果AI评分接口返回错误或超时，系统应有怎样的降级处理方案？
3.  **[邀请/裂变规则]** 当同一个新用户先后通过A和B两个老用户的邀请链接注册时，系统如何判定该新用户的邀请归属？是按首次点击的邀请链接绑定，还是按最后一次点击/注册的链接绑定？如果设备上有缓存或Cookie干扰，如何处理？
4.  **[奖励发放规则]** “评论区随机抽取50名”的具体实现逻辑是什么？是有后台运营手动抽取，还是系统根据规则自动抽取（如需配置抽奖API）？如果是系统自动抽取，抽奖基数是什么（所有在话题区评论过“会诊”的用户？所有评论过的用户？）？
5.  **[图片内容]** 输入材料中包含大量图片引用，由于运行时限制，我（AI）无法分析图片内容。图片中是否包含了测试题目的选项图标（如咖啡杯、揉脖子动画）、用户界面布局（按钮位置、颜色、大小）或具体的互动元素（如评论区如何展示“会诊”卡片）？**这些关键UI和交互细节**需要人工查看设计图并补充给测试团队。
