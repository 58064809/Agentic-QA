# 需求分析草稿

## 1. 需求背景与目标

*   **业务背景**：5月活动“脆皮打工人养护中心”，以“拒绝内卷，在线会诊”为Slogan，通过趣味测试和互动玩法，吸引用户参与并传播，最终为平台职场圈、治愈圈、专业咨询板块引流。
*   **业务目标**：
    1.  吸引用户参与脆皮测试，生成可保存/分享的诊断书，促进活动传播和用户拉新。
    2.  通过活动话题区互动，提升用户活跃度和停留时长。
    3.  通过裂变玩法，带动新用户注册。
    4.  将活动流量导流到平台核心功能模块（职场圈、专业咨询等），实现用户留存。

## 2. 业务范围

*   **范围内**：
    *   脆皮测试答题功能（10道题）。
    *   测试结果计算与诊断书生成。
    *   诊断书保存、分享（生成带平台logo的图片）。
    *   活动话题区互动（发布动态、评论“会诊”）。
    *   裂变玩法（邀请好友，双向有礼）。
    *   导流入口（专业养护通道、圈子入口、主理人联动活动）。
    *   奖励体系（参与奖头像框、互动奖养护礼包）。
*   **范围外**：
    *   “给室友开个方子”活动玩法（仅提及，无具体需求）。
    *   “定时红包活动”（仅提及，无具体需求）。
    *   后台管理功能（如配置题目、奖励、运营人员操作后台等，需求有提及但未提供后台PRD）。
    *   具体的支付、订单或商品交易流程。
    *   与平台现有用户等级、积分体系的深度融合（需确认）。
*   **假设**：
    *   用户均为APP注册用户。
    *   裂变分享所需的基础功能（如生成海报、分享链接）平台已存在。
    *   平台已具备用户发表动态、评论的社交功能，活动话题区基于此进行内容汇总展示。

## 3. 角色与权限

*   **角色列表**：
| 角色 | 定义 | 权限范围 | 备注 |
|---|---|---|---|
| 游客 | 未登录APP的用户 | 无参与权限 | 裂变引导可能要求登录后参与，需确认 |
| 普通用户（老用户） | 参与活动、发起裂变、互动 | 完成测试，获取诊断书，分享；发布互动内容；参与评论；进行邀请 | 核心用户角色 |
| 被邀请新用户 | 通过老用户链接注册的新用户 | 完成测试，发布互动内容 | 完成指定操作后，老用户获得奖励 |
| 运营管理员 | 活动运营人员 | 带头评论互动（官方号）；配置活动奖励；管理话题区内容（假设） | 需求未明确后台功能，此为假设 |
| 平台主理人 | 平台职场、养生等圈子主理人 | 发起茶话会、发布专业建议、在话题区互动 | 导流环节中的关键角色 |

*   **权限差异**：普通用户和运营管理员在互动区可能具有不同标识（如“官方会诊”），需确认。

## 4. 主流程拆解

*   **主流程**：用户参与“脆皮打工人养护中心”活动。
    1.  **入口**：用户通过活动H5页面/APP内Banner/分享链接进入活动主页面。
    2.  **测试答题**：用户按顺序回答10道关于职场日常的单选题。
    3.  **AI评分**：系统根据用户选择，自动计算总分（A=4分, B=3分, C=2分, D=1分）。
    4.  **生成诊断书**：系统根据总分数段（10-15:轻度, 16-25:中度, 26-40:重度），生成包含对应等级、症状、玩的梗、医嘱的诊断书。
    5.  **结果展示与分享**：用户可查看诊断书，并选择生成带平台logo的图片进行保存或分享。
    6.  **发布动态**：用户可在活动话题区发布包含诊断书的动态。
    7.  **互动**：其他用户在该动态下评论“会诊”，发布“药方”。

## 5. 分支流程与异常流程

*   **分支流程**：
    *   **裂变邀请**：用户在生成诊断书后，可发起邀请。老用户分享专属链接/海报给新用户 -> 新用户通过链接注册并参与活动 -> 老用户获得奖励。
    *   **导流互动**：用户在活动页面通过入口进入专业养护通道、相关圈子。
    *   **官方互动**：运营管理员在评论区带头评论。
*   **异常流程**：
    *   **用户取消分享**：用户在生成诊断书后，不选择分享，直接退出。流程中断，测试结果保存（是否需要保存？需确认）。
    *   **网络异常**：答题过程中或生成诊断书时断网。应提示网络异常，已答题目保存状态？需确认。
    *   **服务器错误**：生成诊断书接口或积分接口报错。应展示通用错误页或友好提示。
    *   **分享链接失效**：新用户点击的分享链接过期或无效。应提示“活动已结束”或“链接无效”。
    *   **新用户注册失败**：新用户通过邀请链接注册APP时失败。应向新用户展示注册失败原因，并可能影响老用户奖励发放。
    *   **奖励领取失败**：老用户在满足条件后领取奖励时失败。应有补偿或重试机制。
    *   **高并发场景**：大量用户同时参与活动，导致评分接口或生成诊断书接口响应缓慢或失败。需考虑限流、降级措施。

## 7. 数据字段与状态流转

*   **核心字段**：
    | 实体 | 字段名 | 类型 | 说明 |
    |---|---|---|---|
    | 测试题目 | question_id, order, question_text | 需设计题库表 | 10道题的题目、选项 |
    | 用户测试记录 | test_id, user_id, question_answer_list (JSON) | 记录用户本次测试的完整答题结果 | |
    | 用户测试结果 | result_id, test_id, user_id, score, level, diagnosis_content | 记录用户总分、等级、生成的诊断内容 | |
    | 诊断书图片 | image_id, user_id, result_id, image_url | 生成的诊断书图片URL | 可复现或保存 |
    | 用户邀请记录 | invite_id, inviter_user_id, invitee_user_id, invite_time, status | 记录邀请行为及状态 | status: 已邀请，已注册，已完成 |
    | 奖励记录 | reward_id, user_id, reward_type, reward_status, expire_time | 记录用户获得的奖励 | 头像框、礼包等 |

*   **状态流转**：
    *   **测试流程状态**：未开始 -> 答题中 -> 已完成 (评分完成) -> 已查看诊断书
    *   **邀请流程状态**：已发送邀请 -> 受邀用户未注册 -> 受邀用户已注册 -> 受邀用户未完成任务 -> 受邀用户已完成任务 (触发奖励)

*   **图片生成状态**：
    *   **用户操作**：点击“生成图片” -> 系统请求AI/模版生成服务 -> 图片生成成功/失败 -> 用户看到结果并可选保存/分享。

## 8. 接口与依赖系统

*   **接口建议**：
    *   `POST /api/v1/activities/brittle-worker/submit-test`：提交测试答案并获取分数和诊断结果。
    *   `POST /api/v1/activities/brittle-worker/generate-image`：根据诊断结果生成诊断书图片。
    *   `GET /api/v1/activities/brittle-worker/questions`：获取测试题目列表（当前10题）。
    *   `POST /api/v1/invite/create`：老用户创建邀请链接/海报。
    *   `GET /api/v1/invite/<invite_code>`：新用户点击邀请链接后的落地页接口。
    *   `POST /api/v1/invite/verify`：新用户注册后，校验其是否通过邀请链接注册。
    *   `POST /api/v1/activities/brittle-worker/claim-reward`：用户领取奖励（头像框/礼包）。
*   **依赖系统**：
    *   平台用户中心：用户注册、登录、身份校验。
    *   平台图片服务：用于生成、存储诊断书图片。
    *   平台社交模块：话题区、评论功能的API。
    *   平台奖励中心：用于发放头像框、礼包等奖励。
    *   消息推送服务：用于通知用户活动开始、奖励到账。
    *   AI服务（假设）：如需AI生成诊断书文案或图片，需对接AI服务。

## 11. 待确认问题

1.  **图片内容**（原文中多次出现图片引用但未展示内容）：原文包含大量“症状对应图片”表、诊断书样式等图片引用，当前无法分析图片的具体字段、布局、按钮和交互逻辑。**请人工提供设计稿或说明完成测试后诊断书的展示样式和操作提示。**
2.  **重复参与规则**：用户是否可以多次答题？每次答题后诊断书是否可重置？奖励（如头像框）是否可以重复领取？什么情况下算“已完成测试”？
3.  **新用户注册细节**：新用户通过邀请链接注册后，是否需要完善特定信息（如手机号）才算完成注册？“发布1条互动内容”是指必须发布动态，还是包括评论？互动内容有字数或图片限制吗？
4.  **奖励实现细节**：“脆皮养护礼包”和“一次免费专业养护通道活动”具体是什么形式？名称、内容、有效期、发放规则需要明确。“终极养护礼包”同理。
5.  **导流页面交互**：活动话题区、专业养护通道、圈子入口的具体跳转方式和交互逻辑是什么？用户点击后是跳转到新页面还是打开新标签页？
