# 手机号密码登录需求

## 背景

用户需要通过手机号和密码登录系统。登录成功后，客户端获得 token 并用于后续接口访问。

## 功能范围

- 用户输入手机号和密码后发起登录。
- 手机号格式必须符合中国大陆手机号格式。
- 密码错误时返回明确错误提示，但不泄露账号是否存在等敏感信息。
- 同一账号连续输入错误密码 5 次后锁定 15 分钟。
- 锁定期间再次登录应提示账号暂时锁定，并返回剩余锁定时间。
- 登录成功后返回访问 token、token 类型和过期时间。
- token 过期后，访问受保护资源应提示重新登录。

## 非目标范围

- 本需求不覆盖注册、找回密码、短信验证码登录和多因素认证。
- 本需求不定义具体加密算法实现。

## 验收标准

- 正确手机号和正确密码可以登录成功。
- 错误密码会提示“手机号或密码错误”。
- 连续错误 5 次后账号锁定 15 分钟。
- 登录成功响应中包含 token 和过期时间。
- token 过期后必须重新登录。

## 待人工审核

- 错误提示是否需要统一为更模糊的安全文案。
- 锁定计数是否按账号、手机号、设备或 IP 维度计算。
- token 有效期和刷新策略是否另有安全要求。
