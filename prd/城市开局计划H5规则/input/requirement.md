# 奖金池计算逻辑

## 核心公式

```Plain Text
本场奖金池 = 新玩家人数 × 新玩家单价 + 老玩家人数 × 老玩家单价
```

其中：

- 新玩家单价、老玩家单价由该主理人的 **第几场有效活动** 决定。

- 新老玩家人数以本场活动实际报名/核销数据为准。

- H5 展示阶段可按当前报名人数做预估。

- 最终奖励以活动完成后的核销数据为准。

# 单人预计可领金额计算

## 核心公式

```Plain Text
预计人均奖励 = 本场奖金池 ÷ 预计获奖人数
```

前端展示建议：

```Plain Text
玩一局预计可领 ¥X/人
```

篮球对抗类活动可以展示为：

```Plain Text
胜队预计人均可领 ¥X/人
```

## 金额取整

活动列表h5前端展示建议按整数展示：

```Plain Text
预计人均奖励 = 本场奖金池 ÷ 预计获奖人数
展示时四舍五入取整
```




# 对抗类活动定义

**对抗类活动：指活动参与者在同一场活动中，按照主理人设定的玩法规则分为两个或多个对抗方，通过比赛、积分或排名产生明确结果，并以该结果作为获奖依据的活动。**

满足以下条件的活动，可定义为**对抗类活动**：

1. 活动有明确的对抗关系，比如队伍对队伍、个人对个人、多人积分排名。

2. 活动有明确胜负或排名规则。

3. 活动结束后，能确认获胜队伍、获胜个人或排名靠前用户。

## 常见对抗类活动

# 新玩家 / 老玩家定义

## 新玩家

活动期内，首次通过交子立方 App 报名并完成有效核销的玩家。

## 老玩家

活动期内，此前已完成过有效核销，并再次报名参与活动的玩家。

## 统计口径

同一用户在同一场活动内只能计算一次。

建议按：

```Plain Text
user_id 去重
```

---

# 主理人第几场定义

## 场次判断

```Plain Text
主理人在活动期内完成的第N场有效活动
```

例如：

- 精锋灌篮完成第1场有效活动，则使用第1场奖励标准。

- 精锋灌篮完成第4场有效活动，则使用第4场奖励标准。

- 第10场以后统一按第10场标准计算。

## 有效活动定义

一场活动同时满足以下条件，才算有效活动：

1. 在 App 内发布。

2. 报名人数大于 5 人。

3. 活动实际完成。

4. 有到场核销 / 签到 / 平台确认记录。


```Plain Text

```

可用于：

# 圈子成长金达标规则

## 达标奖励配置

说明：

同一圈子只领取一个最高档位奖励，不叠加。

例如：一个圈子同时满足 1000 元和 1500 元档位，只发放 1500 元成长金。

---

## 参与玩家统计规则

**参与玩家**按去重人数计算，不按人次计算。

判断口径：

同一用户在同一圈子内参加多场有效活动，只算 1 人。

建议统计依据：

用户完成报名并完成到场核销 / 签到 / 平台确认后，计为参与玩家。

---

## 6\.3 内容统计规则

**内容**指圈子内发布的活动相关动态。

内容需满足：

1. 发布在对应圈子内。

2. 内容与该圈子活动相关。

3. 在活动期内发布。

4. 真实有效，不能是灌水、刷量、无关内容。

---

## 6\.4 同档超额排序规则

如果同一档位达标圈子数量超过名额，按以下顺序排序：

1. 参与玩家数更多者优先。

2. 参与玩家数相同，看有效活动数。

3. 有效活动数相同，看内容数。

4. 内容数相同，看达标时间，先达标者优先。
