# 测试用例标准

`testcases` 候选必须使用固定 11 列：

1. 用例ID
2. 需求/规则来源
3. 标题
4. 测试类型
5. 优先级
6. 前置条件
7. 测试数据
8. 测试步骤
9. 预期结果
10. 断言/证据
11. 待确认项

覆盖矩阵必须有表头，并将每条规则或风险映射到实际用例 ID；不接受范围缩写、暂无、待补充等
占位映射。断言必须对应可获取的证据，来源未定义的页面、按钮、日志、后台或数据库观察点只能
标记待确认，不能伪造成已实现能力。

通用质量策略始终校验 11 列、覆盖矩阵、证据真实性与 Schema；命名业务策略在 workspace
显式选择后追加执行。候选写入 `workspaces/<id>/candidates/<run_id>/testcases.md`，人工批准和
确定性 promote 成功后才发布。
