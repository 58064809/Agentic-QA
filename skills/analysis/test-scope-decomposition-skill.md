# 测试范围拆解 Skill

## 元数据

- Skill ID: S4
- 第一版必装: 是
- 适用阶段: Runtime QA Workflow

## 作用

拆模块、功能点、影响范围、正逆向链路

## 输入

需求分析、PRD 正文、接口文档

## 输出

功能模块、影响面、正向链路、逆向链路、回归范围

## 执行步骤

- 按业务对象、页面/接口、角色、状态拆分模块
- 识别范围内、范围外和上下游影响面
- 拆主成功链路、异常链路、权限链路和数据一致性链路
- 输出需求点到测试范围映射

## 质量门

- 输出必须可追溯到 PRD、规则、历史知识或人工确认项。
- 不确定内容必须标记“待确认”“待补充”或“假设”。
- 不得输出纯模板、占位内容或未经确认的正式结论。
