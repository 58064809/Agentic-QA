# 风险识别 Skill

## 元数据

- Skill ID: S5
- 第一版必装: 是
- 适用阶段: Runtime QA Workflow

## 作用

识别高风险：金额、库存、状态流、权限、并发、异常、边界

## 输入

需求规则、状态、字段、接口、历史缺陷

## 输出

风险清单、优先级建议、重点测试方向

## 执行步骤

- 扫描金额、库存、奖励、次数、有效期、权限、状态流等高风险关键词
- 识别并发、重复提交、幂等、弱网、依赖失败和补偿风险
- 识别图片/接口/后台配置缺失导致的不可验证风险
- 把高风险映射到 P0/P1 用例

## 质量门

- 输出必须可追溯到 PRD、规则、历史知识或人工确认项。
- 不确定内容必须标记“待确认”“待补充”或“假设”。
- 不得输出纯模板、占位内容或未经确认的正式结论。
