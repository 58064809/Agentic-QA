# 业务规则提取技能

## 能力说明

从 PRD 和接口文档中提取可测试的业务规则。

## 使用方法

- 搜索限制词：必须、不得、最多、至少、连续、过期、锁定。
- 将每条规则写成条件、动作、结果。
- 把跨接口、跨页面、跨状态的规则单独列出。

## 输出检查

- 每条规则是否可设计测试。
- 规则之间是否存在冲突。

## 限制

安全、合规和风控规则需要人工确认。

## 适用场景

适用于从 PRD、接口文档和验收标准中提取可测试业务规则。

## 输入信息

- 需求文本、接口字段、错误码、状态码、验收标准。

## 输出要求

- 每条规则写成“条件 -> 动作 -> 结果”，并标明来源和待确认项。

## 常见遗漏点

- 漏掉错误提示、安全提示、状态副作用。
- 漏掉跨接口规则，例如登录成功后 token 访问受保护资源。

## 示例

规则：当 token 过期访问受保护资源时，系统应返回 `TOKEN_EXPIRED` 并提示重新登录。
