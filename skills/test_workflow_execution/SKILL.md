# 测试闭环执行 Skill

目标：把需求到执行串成轻量闭环。

执行顺序：

1. 读取当前需求包中的 PRD 和原型信息。
2. 调用用例生成能力产出结构化测试用例。
3. 将用例转换为执行计划，明确 case 到 API/page/fixture/DB 的绑定状态。
4. 自动生成测试环境模板、测试数据模板和 pytest 合约脚本。
5. 执行 pytest。
6. 分析 pytest 结果并保存输出。

约束：

- 不允许把未绑定真实动作的自然语言用例伪装成通过。
- 未配置环境或未绑定执行器时，pytest 用例必须 skip，并说明缺少什么。
- 已存在的手写/已绑定 pytest 用例可以一起执行。
