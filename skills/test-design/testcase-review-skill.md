# 用例评审 Skill

## 元数据

- Skill ID: S8
- 第一版必装: 是
- 适用阶段: Runtime QA Workflow

## 作用

检查漏测、重复、预期不清、步骤不完整、优先级错误

## 输入

测试用例草稿、需求分析、规则、风险清单

## 输出

质量错误、warnings、人工评审点

## 执行步骤

- 检查表头、列数、优先级、用例数量和占位内容
- 检查需求/规则来源、测试数据、断言/证据是否空泛
- 检查主流程、异常、边界、权限、状态、幂等、数据一致性、回归是否覆盖
- 把可修复问题阻断，把需人工补充的问题标记 warning

## 质量门

- 输出必须可追溯到 PRD、规则、历史知识或人工确认项。
- 不确定内容必须标记“待确认”“待补充”或“假设”。
- 不得输出纯模板、占位内容或未经确认的正式结论。
