# 边界值分析

## 概述

围绕输入域的上限、下限、触发阈值和边界条件设计测试用例，验证系统在精确边界处的行为。

适用于次数限制、时长窗口、长度限制、金额范围、分页大小、年龄段等离散或连续边界明确的规则。

## 适用时机

- 存在数值上限/下限（如 N 次、M 分钟、X 字）
- 存在状态触发阈值（如第 N 次触发锁定）
- 等价类划分完成后补充边界精度

## 前置知识

- 了解等价类划分的基本概念
- 确认边界值的精度单位（如"次"是整数，"分钟"是秒级？）

## 操作步骤

### Step 1: 识别所有边界

从 PRD 和接口文档中提取所有带数字的约束：

| 边界类型 | 示例 |
|---------|------|
| 次数边界 | 连续输错 5 次锁定 |
| 时长边界 | 锁定 15 分钟、token 过期 30 分钟 |
| 长度边界 | 密码 6-20 位 |
| 金额边界 | 单笔限额 5000 元 |
| 分页边界 | 每页最多 100 条 |

### Step 2: 确定边界值

以二值边界法（2-value）为标准，特殊场景用三值边界法（3-value）：

| 方法 | 取值 | 适用场景 |
|------|------|---------|
| 二值边界 | N-1, N | 连续范围（如金额 0 ~ 5000） |
| 三值边界 | N-1, N, N+1 | 离散次数（如第 5 次触发锁定） |

### Step 3: 设计预期结果

每个边界值必须明确预期行为：

| 取值 | 预期 |
|------|------|
| N-1 | 正常处理（未触发边界条件） |
| N   | 触发边界处理（如锁定、拒绝、截断） |
| N+1 | 触发的持续状态（如果适用） |

### Step 4: 补充恢复边界

对有时间窗口/恢复条件的边界，设计恢复前后的状态序列。

### 边界值分析表

| 规则 | 边界类型 | N 值 | N-1 | N | N+1 | N-1 预期 | N 预期 | N+1 预期 |

| 连续输错5次锁定 | 次数 | 5 | 4 | 5 | 6 | 正常计数 | 触发锁定 | 提示已锁定 |

## 自检清单

| 检查项 | 通过标准 | 自查 |
|--------|----------|:----:|
| 上下界覆盖 | N-1 和 N 已覆盖（二值），必要时覆盖 N+1（三值） | □ |
| 恢复边界 | 锁定/过期后的恢复条件已测试 | □ |
| 精度匹配 | 边界值的精度单位与实际业务一致（如分钟 vs 秒） | □ |
| 预期绑定 | 每个边界值已绑定明确预期结果 | □ |
| 状态序列 | 边界触发的后续操作（锁定后/过期后）已有用例 | □ |

## 常见误区

- ❌ 忽略 N-1（不是边界）和 N+1（超出边界）的覆盖
- ❌ 漏测边界触发后的下一次操作（如第 6 次操作）
- ❌ 边界值选错精度（如锁定时长 15 分钟只测 15，不测 899 秒 / 901 秒）

## FAQ

**Q: 二值边界和三值边界怎么选？**
A: 连续范围（数值、金额、年龄）用二值。离散次数、触发阈值用三值。顺序是：三值提供的信息多于二值，但用例量也多 50%，根据风险决定。

**Q: 边界值分析和等价类划分的顺序？**
A: 先划分等价类，再用边界值分析补充等价类的边界。等价类负责"覆盖面"，边界值负责"精度"。
