# 测试用例生成 Skill

## 元数据

- Skill ID: S7
- 第一版必装: 是
- 适用阶段: Runtime QA Workflow

## 作用

按固定格式生成测试用例

## 输入

需求分析、测试范围、风险清单、测试方法矩阵

## 输出

Markdown 富表格测试用例草稿

## 执行步骤

- 生成 11 列富表格：用例ID、来源、标题、类型、优先级、前置、数据、步骤、预期、证据、待确认
- 优先生成 P0 主链路、核心规则、权限、状态、边界和奖励/库存/金额风险用例
- 每条用例必须有具体测试数据和至少两类断言证据
- 信息不足时写具体待确认项，不输出占位

## 质量门

- 输出必须可追溯到 PRD、规则、历史知识或人工确认项。
- 不确定内容必须标记“待确认”“待补充”或“假设”。
- 不得输出纯模板、占位内容或未经确认的正式结论。
