# API 契约分析技能

## 能力说明

解析接口方法、路径、请求、响应、错误码、鉴权和幂等要求。

## 使用方法

- 对每个接口建立字段和约束表。
- 区分 HTTP 状态码和业务码。
- 标记缺失、冲突或不一致的接口信息。
- 将契约映射为 API 断言。

## 输出检查

- 请求和响应是否完整。
- 错误场景是否覆盖。

## 限制

接口文档与实现不一致时不能自行认定，以证据和人工确认为准。

## 适用场景

适用于生成 API 测试计划和 pytest 脚本前的接口契约拆解。

## 输入信息

- 方法、路径、请求体、响应体、错误码、鉴权、幂等和限流规则。

## 输出要求

- 输出字段约束、断言点、错误场景和待确认接口问题。

## 常见遗漏点

- 混淆 HTTP 状态码和业务码。
- 忽略错误响应结构和鉴权失败。

## 示例

`POST /api/v1/auth/login` 应断言 200 成功结构、401 密码错误、423 账号锁定。
