# 需求理解 Skill

## 元数据

- Skill ID: S1
- 第一版必装: 是
- 适用阶段: Runtime QA Workflow

## 作用

解析 PRD / Markdown / 飞书文档，提取背景、角色、流程、规则、限制

## 输入

input/requirement.md, input/api.md, workspace.yml

## 输出

背景、目标、角色、主流程、分支流程、业务规则、限制、待确认项

## 执行步骤

- 识别需求背景、业务目标和上线范围
- 抽取用户角色、权限、状态和核心对象
- 拆出主流程、分支流程、异常流程
- 标记图片/原型/接口缺口，不猜测未给信息

## 质量门

- 输出必须可追溯到 PRD、规则、历史知识或人工确认项。
- 不确定内容必须标记“待确认”“待补充”或“假设”。
- 不得输出纯模板、占位内容或未经确认的正式结论。
