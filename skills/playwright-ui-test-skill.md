# Playwright UI 测试技能

## 能力说明

用 Playwright 生成稳定 UI 自动化草稿。

## 使用方法

- 优先使用可访问名称、role、label、test id。
- 对异步页面使用明确等待。
- 每个测试只覆盖一个关键目标。
- 失败时保留截图、trace 或日志策略。

## 输出检查

- 选择器是否稳定。
- 断言是否反映业务结果。

## 限制

对验证码、短信和第三方跳转必须标注人工或模拟策略。

## 适用场景

适用于把关键用户路径转换为 Playwright UI 自动化草稿。

## 输入信息

- 页面 URL、用户路径、选择器策略、测试账号、环境说明。

## 输出要求

- 输出脚本草稿、选择器策略、断言点、等待策略和不可自动化说明。

## 常见遗漏点

- 使用脆弱 CSS 层级选择器。
- 缺少截图、trace 或失败诊断策略。

## 示例

登录按钮应优先使用 role 或测试 ID，而不是 `.btn:nth-child(2)`。
