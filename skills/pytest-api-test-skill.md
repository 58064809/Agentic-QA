# pytest API 测试技能

## 能力说明

用 pytest 组织 API 自动化测试。

## 使用方法

- 使用 fixture 管理 base_url、认证和测试数据。
- 使用参数化覆盖等价类和边界值。
- 使用明确断言校验状态码、业务码和字段。
- 通过环境变量传入敏感配置。

## 输出检查

- 脚本能被 pytest 收集。
- 失败信息可定位。

## 限制

不生成真实生产数据和密钥。

## 适用场景

适用于把接口契约和用例转成 pytest 草稿。

## 输入信息

- API 测试计划、接口文档、测试数据、环境变量约定。

## 输出要求

- 输出 pytest 文件、fixture、参数化用例、skip 保护和断言说明。

## 常见遗漏点

- 默认请求生产地址。
- 缺少环境变量时测试失败而不是 skip。
- 断言只检查 HTTP 200。

## 示例

没有 `LOGIN_API_BASE_URL` 时，登录 API 草稿测试应 `pytest.skip`，避免误连真实环境。
