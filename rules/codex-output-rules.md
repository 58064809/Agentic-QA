# Codex 输出规则

## 适用范围

本规则适用于 Codex 在完成仓库任务后的 Chat 回复、阶段性说明和验收总结。

## 输出原则

- 不在 Chat 中粘贴完整大文件。
- 不在 Chat 中粘贴完整 diff。
- 大内容必须写入仓库文件，并在回复中给出文件路径。
- 单次回复应保持可读，优先输出摘要、关键路径、验收结果和待人工确认项。
- 若任务产生大量文件，按模块分组说明，不逐段复述文件内容。
- 若用户明确要求查看长内容，优先提示对应文件路径，并只摘录必要片段。

## 完成任务后的推荐结构

1. 变更摘要：说明改了什么，不复述完整文件。
2. 关键文件：列出最重要的产物路径。
3. 验收结果：列出已执行命令及是否通过。
4. 待人工确认：列出必须由人确认的业务结论、审核门或环境授权。

## 禁止事项

- 不把完整 Markdown 产物粘贴到回复中。
- 不把完整代码 diff 粘贴到回复中。
- 不用长篇解释替代仓库内的正式文档。
- 不输出未经人工确认的正式 QA 结论。
