# 自动化规则

- 优先使用成熟工具：pytest 用于 API 和服务端测试，Playwright 用于 UI 测试。
- 只生成可读、可维护、可审查的脚本。
- 配置和敏感数据通过环境变量或本地配置传入，不提交密钥。
- 脚本必须可以独立执行，并说明依赖环境。
- 生成脚本后默认进入人工审核，不得直接视为可信产物。

## 安全边界

- 默认禁止连接生产环境；只有用户明确给出非生产或授权环境才可执行。
- 禁止提交真实手机号、密码、token、cookie、密钥和生产域名。
- 自动化脚本必须允许通过环境变量注入 base URL 和测试账号。
- 没有必要环境变量时，pytest 脚本应 skip，不应失败。
- 自动化生成产物必须标记为草稿，等待人工审核后才能进入执行。
