# 等价类划分

## 概述

将输入数据划分为有效等价类（合法输入）和无效等价类（非法输入），每个等价类选一个代表值，用最少用例覆盖最多场景。

适用于手机号、密码、金额、枚举值、状态码等输入空间较大的字段。

## 适用时机

- 字段有明确取值范围或格式约束
- 输入组合爆炸，需要降维
- 测试设计初期，作为第一层覆盖

## 前置知识

- 了解被测字段的业务约束文档（PRD / API 文档）
- 能区分合法值、非法值、边界值的概念

## 操作步骤

### Step 1: 列出所有输入字段

对每个被测接口/页面，列出所有输入字段。

### Step 2: 确定每个字段的约束

| 维度 | 检查内容 |
|------|----------|
| 类型 | string / int / enum / boolean / object |
| 长度 | 最小长度、最大长度 |
| 格式 | 正则、枚举值、编码规则 |
| 范围 | 数值最小值、最大值 |
| 可选性 | 必填 / 选填 / 有条件必填 |

### Step 3: 划分等价类

对每个字段，建立三类等价类：

| 等价类类型 | 说明 | 示例（手机号字段） |
|-----------|------|-------------------|
| 有效等价类 | 完全合法的输入 | 11 位中国大陆手机号 |
| 无效等价类 | 明确非法的输入 | 少于 11 位、含字母、空值 |
| 特殊等价类 | 边界附近或格式合规但业务不合规 | 非大陆号段、靓号段 |

### Step 4: 选取代表值

每个等价类选一个典型代表值，说明选取理由。

## 自检清单

| 检查项 | 通过标准 | 自查 |
|--------|----------|:----:|
| 有效等价类覆盖 | 每个合法路径都有代表值 | □ |
| 无效等价类覆盖 | 每种约束违规都有代表值（长度/格式/类型/空值） | □ |
| 特殊等价类 | 边界值、异常格式、权限不足已覆盖 | □ |
| 代表值合理性 | 代表值在真实场景存在（如号段可用） | □ |
| 避免冗余 | 同一等价类不重复选取多个代表值 | □ |

## 常见误区

- ❌ 只测有效等价类，忽略无效等价类
- ❌ 把边界值当成等价类代表值（边界值用边界值分析处理）
- ❌ 忽略空值、空白字符、JSON 类型错误等"隐式"无效类

## FAQ

**Q: 等价类划分和边界值分析的区别？**
A: 等价类划分解决"选哪个代表值"，边界值分析解决"边界附近是否精准"。通常先用等价类确定大范围，再用边界值分析深入边界。

**Q: 枚举类型字段如何划分？**
A: 将每个枚举值视为一个等价类。如果枚举值较多（>10），可按行为相似度归类，如"正常状态值"和"终态值"。
