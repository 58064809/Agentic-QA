# 状态迁移建模

## 概述

将业务对象的状态、事件和迁移规则转化为测试模型，系统化覆盖所有合法的与非法的状态转移路径。

适用于账号锁定、订单审批、token 生命周期、审批流、任务状态等状态驱动的业务规则。

## 适用时机

- 业务对象有明确的有限状态集
- 状态迁移的条件（守卫条件）可枚举
- 需要测试非法/不可达迁移

## 前置知识

- 能建立状态机模型：状态 × 事件 → 新状态
- 理解"守卫条件"（guard condition）约束状态迁移

## 操作步骤

### Step 1: 枚举所有状态

| 状态 | 说明 | 是否初始 | 是否终止 |
|------|------|:--------:|:--------:|
| NORMAL | 账号正常 | ✓ | |
| ERROR_COUNTING | 错误计数中 | | |
| LOCKED | 已锁定 | | ✓ |
| UNLOCKED | 已解锁恢复 | | |

### Step 2: 枚举所有事件和守卫条件

| 事件 | 触发条件 | 守卫条件 |
|------|---------|---------|
| 密码错误 | 用户输入错误密码 | 当前状态 != LOCKED |
| 登录成功 | 用户输入正确密码 | 当前状态 = NORMAL |
| 时间到期 | 锁定时间达到 15 分钟 | 当前状态 = LOCKED |

### Step 3: 构建状态迁移表

| 当前状态 | 事件 | 守卫条件 | 下一状态 |
|---------|------|---------|---------|
| NORMAL | 密码错误 | count < 5 | ERROR_COUNTING |
| ERROR_COUNTING | 密码错误 | count < 5 | ERROR_COUNTING |
| ERROR_COUNTING | 密码错误 | count == 5 | LOCKED |
| LOCKED | 时间到期 | 超时 >= 15min | UNLOCKED |

### Step 4: 设计测试路径

| 路径类型 | 说明 | 示例 |
|---------|------|------|
| 合法主路径 | 正常业务流程 | NORMAL → (登录成功) → NORMAL |
| 合法异常路径 | 触发守卫后恢复 | NORMAL → 连错 4 次 → 第 5 次 → LOCKED → 到期 → UNLOCKED |
| 非法迁移 | 不允许的操作 | 在 LOCKED 状态下登录成功 |

## 输出模板

### 状态迁移图（文本）

```
[NORMAL] ──密码错误(0→1)──▶ [ERROR_COUNTING] ──密码错误(4→5)──▶ [LOCKED]
    │                            │                           │
    ├──登录成功─────────────────▶ NORMAL                     │
    │                            └──登录成功────────────────▶ NORMAL
    │                                                        └──时间到期──▶ [UNLOCKED]
```

### 状态迁移表

| # | 当前状态 | 事件 | 守卫 | 下一状态 | 预期行为 |
|:-:|---------|------|------|---------|---------|
| 1 | NORMAL | 密码错误 | - | ERROR_COUNTING | 提示错误，剩余 4 次 |
| 2 | ERROR_COUNTING | 密码错误 (count=4) | count < 5 | ERROR_COUNTING | 提示错误，剩余 1 次 |
| 3 | ERROR_COUNTING | 密码错误 (count=5) | count == 5 | LOCKED | 触发锁定，提示 15 分钟后重试 |

## 自检清单

| 检查项 | 通过标准 | 自查 |
|--------|----------|:----:|
| 状态完备 | 所有可达状态已列出，含初始和终止 | □ |
| 事件完备 | 所有触发状态变更的事件已列出 | □ |
| 守卫明确 | 每个迁移的守卫条件已写明 | □ |
| 合法迁移 | 正常业务路径已覆盖 | □ |
| 非法迁移 | 不允许的状态跳转已测试 | □ |
| 重复触发 | 同一事件的重复触发已测试 | □ |

## 常见误区

- ❌ 漏掉初始状态和终止状态
- ❌ 忽略非法迁移（系统应拒绝而不应进入未定义状态）
- ❌ 守卫条件简化（如只写"超时"不写具体阈值）

## FAQ

**Q: 状态迁移表和场景建模的区别？**
A: 状态迁移关注**业务对象的内置状态变化**；场景建模关注**用户的端到端操作路径**。状态迁移是"对象视角"，场景建模是"用户视角"。

**Q: 状态太多怎么办？**
A: 优先覆盖核心业务对象（如订单、账号、审批单）。辅助对象（如日志、通知）可降级为场景建模中覆盖。
